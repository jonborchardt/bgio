# 07.7 — Foreign UI panel

**Parent:** [07-foreign.md](07-foreign.md)

## Goal

A panel showing army composition, both decks (face-down with counts), the
in-flight battle if any, and the recruit / flip / assign-damage controls.

## Files

- Create `src/ui/foreign/ForeignPanel.tsx`.
- Create `src/ui/foreign/Army.tsx`.
- Create `src/ui/foreign/Decks.tsx`.
- Create `src/ui/foreign/BattlePanel.tsx`.
- Create `src/ui/foreign/AssignDamageDialog.tsx`.
- Modify `src/Board.tsx` to render `<ForeignPanel>` for the seat holding
  Foreign.
- Create `tests/ui/foreign/ForeignPanel.test.tsx`.

## API

```tsx
export const ForeignPanel: React.FC<BoardProps<SettlementState>>;
```

## Steps

1. `<Army>`: list `<UnitCard count={n}>` from 09.2 for each `inPlay` entry.
   Click → release dialog.
2. `<Hand>` (reuse the Domestic hand pattern): drag a hand `<UnitCard>` to
   the army row → `foreignRecruit`.
3. `<Decks>`: two `<DeckStack>` from 09.5; click "Flip Battle" →
   `foreignFlipBattle`. Disabled if a battle is in-flight or after a loss
   ended the foreign stage.
4. `<BattlePanel>` shows the in-flight card and the resolver preview
   (run resolveBattle with current allocations to mirror live result).
5. `<AssignDamageDialog>` — modal: per-round damage absorber chooser. On
   submit → `foreignAssignDamage(allocations)`.

## Tests

- Recruit click fires the move with correct args.
- Flip button disables during in-flight.
- Assign-damage dialog refuses submission if any round under-allocates;
  shows the validation error from the resolver.

## Dependencies

07.1, 07.2, 07.3, 07.4, 09.2 (UnitCard), 09.5 (DeckStack).

## Out of scope

- Animated battle replay. Static log is enough for V1.
