# 14.6 — Phase / move hints

**Parent:** [14-playtest-followups.md](14-playtest-followups.md)

## Goal

Show a one-line "what you can do now" hint under the StatusBar so a
new player isn't staring at an unlabeled chief panel wondering what
"chiefPhase" means. Drives off the (phase, stage, role) triple —
see playtest issue V5.

## Files

- Create `src/ui/layout/PhaseHint.tsx`.
- Modify `src/Board.tsx` — render under StatusBar.
- Create `tests/ui/layout/PhaseHint.test.tsx`.

## API

```tsx
export interface PhaseHintProps {
  phase: string | null;
  stage?: string;            // ctx.activePlayers?.[playerID]
  rolesAtSeat: ReadonlyArray<Role>;
}
export const PhaseHint: React.FC<PhaseHintProps>;
```

## Steps

1. Resolve a hint string from a small lookup table:
   - `chiefPhase` + chief role → "Distribute resources to non-chief
     seats. End your turn when ready."
   - `othersPhase` + `scienceTurn` + science → "Pour your wallet
     into a science card. Complete one when paid covers cost."
   - `othersPhase` + `domesticTurn` + domestic → "Buy a building,
     produce, or end your turn."
   - `othersPhase` + `foreignTurn` + foreign → "Pay upkeep, recruit,
     flip a battle, then end your turn."
   - `othersPhase` + `foreignAwaitingDamage` + foreign → "Allocate
     incoming damage."
   - `othersPhase` + stage `done` → "Waiting for the other roles."
   - `endOfRound` → "Round ending — opponent acts now."
   - `playingEvent` → "Resolve the event you played."
2. Spectator (playerID === null) → "Watching."
3. Render as a small Typography under StatusBar; muted color from
   theme tokens.

## Tests

- Each (phase, stage, role) tuple resolves to the expected string.
- Spectator branch.
- Default to empty string when no rule matches (no broken UI).

## Dependencies

09.1 (StatusBar / Board layout), 14.1 (seat picker — needed so the
local viewer's stage can flip while testing).

## Out of scope

- Localization. Strings are inline English.
- Per-card help (hover-tip on individual buttons).
