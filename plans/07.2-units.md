## Goal

Track recruited units efficiently (count-based, not duplicate cards), and
implement the recruit / upkeep / release moves.

## API

```ts
export interface UnitInstance {
  defID: string;       // matches UnitDef.name
  count: number;       // multiples handled here, not by duplicate entries
}

export interface ForeignState {
  hand: UnitDef[];
  inPlay: UnitInstance[];
  battleDeck: BattleCardDef[];
  tradeDeck: TradeCardDef[];
  inFlight: { battle: BattleCardDef | null; committed: UnitInstance[] };
}

export const foreignRecruit: Move<SettlementState>;
// args: defID: string, count?: number (default 1)

export const foreignUpkeep: Move<SettlementState>;
// no args; must be called once at start of foreign's stage

export const foreignReleaseUnit: Move<SettlementState>;
// args: defID: string, count?: number
```

## Steps

1. Hand starts with all level-0 Militia unit cards (per game-design.md Â§Setup
   Foreign). For V1 Militia is identified by a `level: 0` field once unit
   data has it; until then, treat the first 3 cards in `UNITS` as Militia.
2. **Recruit:**
   - Validate caller seat holds `foreign`, in stage `foreignTurn`.
   - Look up `UnitDef`. Cost = `{ gold: def.cost }`.
   - Domestic Forge effect (`unitCost: -1`) reduces gold cost; resolved by
     summing `BenefitEffect` of kind `unitCost` across in-play buildings.
   - Validate affordability; transfer wallet â†’ bank.
   - Increment `inPlay[defID].count` (or add new entry).
3. **Upkeep:**
   - Compute total maintenance: per-unit `def.cost` (V1 stub) reduced by
     building effects (`unitMaintenance`).
   - If wallet covers it: transfer to bank.
   - If not: move stays valid (alternative is `foreignReleaseUnit`); if
     player explicitly chooses to fail to pay, the simplest model is the
     unit released for its cost (returned to bank as gold). For V1, we
     require the player to call `foreignReleaseUnit` for any units they
     can't afford; if upkeep is uncoverable and no release is queued,
     the move fails â€” caller must release first.
4. **Release:** decrement `inPlay[defID].count`; refund half-cost to bank
   per "release value" (V1 default = floor(cost/2)).

## Tests

- Recruit a Brute (`cost 3`): wallet -3, bank +3, `inPlay[Brute].count = 1`.
- Recruit twice: count goes to 2, no duplicate `UnitInstance`.
- Forge in Domestic grid: unit cost -1.
- Upkeep with sufficient wallet: deducts and continues.
- Upkeep with insufficient wallet â†’ `INVALID_MOVE` (must release first).
- Release returns floor(cost/2) to bank, decrements count.
