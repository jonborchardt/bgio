# 02.3 — Randomness

**Parent:** [02-engine.md](02-engine.md)

## Goal

All randomness goes through bgio's random plugin. Same seed → same game.
Manual `Math.random` is forbidden in `src/`.

## Files

- Create `src/game/random.ts` (replaces the temporary helper from 01.3).
- Modify `src/game/index.ts` to enable the random plugin.
- Modify `eslint.config.js` to ban `Math.random`.
- Create `tests/randomness.test.ts`.

## API

```ts
// src/game/random.ts — thin wrappers; bgio gives us these via ({ random })
// inside setup / moves. We re-export typed helpers so call sites read well.

export interface RandomAPI {
  shuffle<T>(arr: ReadonlyArray<T>): T[];   // wraps random.Shuffle
  pickOne<T>(arr: ReadonlyArray<T>): T;     // wraps shuffle + take
  rangeInt(loInc: number, hiExc: number): number; // wraps random.D6 / Number
}

export const fromBgio = (random: any): RandomAPI;
```

## Steps

1. bgio's `Game` config supports the random plugin out of the box; just
   pass `({ random }) => ...` to `setup`/moves and use `random.Shuffle`,
   `random.Number`, `random.D6` etc.
2. `fromBgio` adapts the bgio random object to our `RandomAPI` so move
   bodies read cleanly: `const r = fromBgio(random); r.shuffle(deck);`.
3. ESLint:
   ```js
   'no-restricted-properties': ['error', { object: 'Math', property: 'random',
     message: 'Use the bgio random plugin (src/game/random.ts).' }],
   ```
   Allow it in `tests/` if a test really needs it (none should).
4. Delete the local `shuffle` from 01.3.

## Tests

- Two clients with the same `seed` produce identical state after the same
  move sequence.
- Different seeds produce different shuffles (>= 1 of 100 draws differ).
- ESLint flags a contrived `Math.random()` call in a fixture file.

## Dependencies

02.1 (we typically only call random inside `setup` and moves, both of which
bgio already provides the random API to).

## Out of scope

- Cryptographic randomness. Not needed; bgio's seedable PRNG is fine.
