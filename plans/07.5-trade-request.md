# 07.5 — Trade-request placement & chief-discard interrupt

**Parent:** [07-foreign.md](07-foreign.md)

## Goal

When Foreign flips a trade request, place it in the mat's `tradeRequest`
slot. If a request already exists, enter the `awaitingChiefDecision` stage
so the chief picks which to keep — game-design.md §Foreign:
"You can only have 1 trade request at a time, if collecting a second, the
chief decides which to discard."

## Files

- Create `src/game/roles/foreign/tradeRequest.ts`.
- Create `src/game/roles/chief/decideTradeDiscard.ts` (lives under chief
  because the chief is the one acting).
- Register moves in `src/game/moves.ts`.
- Create `tests/roles/foreign/tradeRequest.test.ts`.

## API

```ts
// Used internally by 07.4's foreignFlipTrade move when a trade is drawn.
export const placeOrInterruptTrade: (
  G: SettlementState, ctx: Ctx, drawn: TradeCardDef,
) => void;

export const chiefDecideTradeDiscard: Move<SettlementState>;
// args: keep: 'existing' | 'new'
```

## Steps

1. `placeOrInterruptTrade`:
   - If `mat.tradeRequest === null`: set it from `drawn`.
   - Else: stash `drawn` in `G.foreign.pendingTrade` and call
     `enterEventStage(events, 'foreignTurn')`-style helper to put the chief
     into stage `awaitingChiefDecision`.
2. `chiefDecideTradeDiscard`:
   - Validate caller holds chief, stage `awaitingChiefDecision`.
   - Validate `keep ∈ {'existing','new'}`.
   - Apply: clear `pendingTrade`; either keep current or replace mat slot.
   - **Refund discarded card's resources** if any were placed on it. Per
     game-design.md "Goods on the discarded card are returned to the bank."
   - Exit interrupt; chief stage returns to `done`; foreign returns to
     `foreignTurn`.

## Tests

- First trade: lands in mat slot, no interrupt.
- Second trade: triggers chief interrupt; mat slot unchanged until decision.
- Chief picks `'new'`: existing's deposited resources refunded to bank;
  new card goes into mat.
- Chief picks `'existing'`: pending discarded; new card's resources (none
  yet, since just drawn) go to bank.
- Resource conservation across the swap.

## Dependencies

02.2, 03.3, 07.4.

## Out of scope

- Other players fulfilling the trade request — that uses the standard
  circle pull mechanic from 03.4.
