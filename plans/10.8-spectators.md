# 10.8 — Spectators

**Parent:** [10-multiplayer.md](10-multiplayer.md)

## Goal

Allow any logged-in user to watch a match in progress as a non-seat
viewer. They see the redacted public state (per `playerView` with
`playerID === null`), can read public chat, but cannot move.

## Files

- Modify `src/lobby/MatchList.tsx` — add a "Watch" affordance per match.
- Create `src/lobby/spectatorClient.ts` — a `Client` factory configured
  without a `playerID`.
- Modify `src/Board.tsx` — render in read-only mode when `playerID ===
  null` (no move buttons; panels collapsed to summary view).
- Modify `server/index.ts` to allow a websocket connection without a
  match-credential — but reject any move dispatch.
- Create `tests/lobby/spectator.test.ts`.

## API

```ts
export const joinAsSpectator: (
  serverUrl: string, matchID: string, authToken: string,
) => Client;
```

## Steps

1. Lobby UI: each `MatchList` row has "Join (seat …)" buttons + a "Watch"
   button. Clicking "Watch" mounts `<App>` with no `playerID`.
2. `<Board>` reads `props.playerID`. When null:
   - All move-emitting buttons hidden.
   - Per-role panels render the read-only summary view from 09.1.
   - Status bar shows "Spectating".
3. Server-side: bgio's transport already handles `playerID = null`
   correctly (returns `playerView(G, ctx, null)`); we just verify it
   compiles for our `playerView` from 02.4.
4. Spectator can chat (10.5) — public chat only.
5. The lobby shows a spectator count per match (`<N watching>`).

## Tests

- A spectator client receives state updates as moves are made.
- A spectator's attempt to dispatch a move is rejected by the server.
- `playerID === null` view: Foreign deck order is fully redacted; bank
  is visible; chat is visible.
- Spectator can leave and rejoin without affecting seats.

## Dependencies

02.4 (playerView), 10.1, 10.3, 10.5, 10.7 (auth gate).

## Out of scope

- Per-team spectators (e.g. "watch only what Foreign sees"). Reserve for
  future "coaching" feature.
- Spectator-only chat channel.
