# 01.3 — Settlement game skeleton

**Parent:** [01-foundations.md](01-foundations.md)

## Goal

Replace the Card Sweep game definition with a typed `Settlement` game whose
only legal move is `pass` — so the engine boots and tests run while real
moves get added in later stages.

## Files

- Modify `src/game.ts` → split into:
  - `src/game/index.ts` — exports `Settlement` and the state types.
  - `src/game/setup.ts` — pure `setup` function.
  - `src/game/moves.ts` — `pass` for now; later moves register here.
- Modify `src/App.tsx` — use `Settlement` instead of `CardSweep`.
- Modify `src/Board.tsx` — render `roleAssignments` and an "End my turn"
  button bound to `pass`.
- Modify `index.html` — title `Settlement` (placeholder).
- Modify `tests/game.test.ts` — keep one smoke test that boots the new
  game; full Card Sweep tests deleted.
- Modify [README.md](../README.md) — note "work in progress, see /plans".

## API

```ts
// src/game/index.ts
import type { Game } from 'boardgame.io';
import type { SettlementState } from './types';

export const Settlement: Game<SettlementState> = { /* ... */ };
export type { SettlementState };
```

## Steps

1. Move the existing `shuffle` helper into `src/game/random.ts` for reuse;
   delete it once 02.3 supersedes it with the bgio random plugin.
2. `setup({ random, ctx })` returns a `SettlementState` with role
   assignments derived from `ctx.numPlayers` via `assignRoles` (01.1).
3. Add a `pass` move that does nothing visible; just lets the harness
   advance turns. It will be removed once real moves exist.
4. `Settlement` config: `name: 'settlement'`, `setup`, `moves: { pass }`,
   `numPlayers: 4` default in `App.tsx`. The local seat is `'0'`; the other
   three seats are filled by bots once 11.x lands. Until bots exist, the
   dev experience is hot-seat — but the default *config* is 4-player so
   the engine is exercised in its full shape from day one.
5. Update `App.tsx` to import `Settlement`. Remove the Card Sweep board
   props type aliasing.
6. `Board.tsx` becomes a stub that lists role assignments and shows whose
   turn it is.

## Tests

- `import { Settlement } from '../src/game'` and boot a 2-player headless
  client; assert the initial state has `roleAssignments` matching `assignRoles(2)`.
- Calling `moves.pass()` advances `ctx.currentPlayer` from `'0'` to `'1'`.

## Dependencies

01.1 (state shape), 01.2 (data loaders — `setup` may seed decks from data
later, but does not have to here).

## Out of scope

- Phases. `setup` produces a flat single-phase game; 02.1 adds phases.
- The deletion of Card Sweep test coverage beyond the one renamed smoke
  test. Card Sweep itself is gone.
