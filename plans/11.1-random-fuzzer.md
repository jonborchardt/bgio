# 11.1 — RandomBot fuzzer

**Parent:** [11-ai.md](11-ai.md)

## Goal

A headless harness that runs many games of bgio's `RandomBot` against itself
and asserts engine invariants. Cheap regression coverage that scales as the
move surface grows.

## Files

- Create `tests/fuzz/randomBot.test.ts`.
- Create `tests/fuzz/invariants.ts` — reusable assertions.

## API

```ts
// tests/fuzz/invariants.ts
export const assertNoNegativeResources: (G: SettlementState) => void;
export const assertConservation: (Gs: ReadonlyArray<SettlementState>) => void;
export const assertTurnsBounded: (state: { ctx: Ctx }, max: number) => void;
```

## Steps

1. Use `Client` from `boardgame.io/client` in headless mode with the
   `RandomBot` driving every seat.
2. **Termination:** the engine ends games at `TURN_CAP` (08.5, default
   80) so every fuzz run terminates naturally. The fuzzer adds a
   secondary safety cap on *moves* (`MAX_MOVES`, default 10000) for
   pathological cases where a turn doesn't advance — those are bugs and
   should fail the run.
3. Capture every state snapshot. After the run:
   - `assertNoNegativeResources` over every snapshot.
   - `assertConservation` between every consecutive pair (delta must come
     from a known source: bank ↔ wallet ↔ circle ↔ paid-card ledger).
   - `assertTurnsBounded(_, MAX_MOVES)` — sanity bound, not a win
     requirement.
4. Default loop: 100 games per CI run, 4 numPlayers configs (1, 2, 3, 4) ×
   25 seeds each. The pass rate metric is "no invariant violations", not
   "every game won". Most random runs end with `timeUp` — that's fine.

## Tests

- The fuzz suite itself doesn't have unit tests; *it is* the test. CI
  treats failure as a regression.
- Each invariant has its own minimal reproducer test (e.g. construct a
  state with -1 gold, assert `assertNoNegativeResources` throws).

## Dependencies

01–08 (more landed = better coverage).

## Out of scope

- Smart bots (11.3–11.6).
- Performance — a 100-game run should land under ~10s; if it doesn't,
  add a sampling sub-plan.
