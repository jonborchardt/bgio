# 09.5 — Deck stacks & hand layouts

**Parent:** [09-ui.md](09-ui.md)

## Goal

Two reusable layout primitives: `<DeckStack>` (face-down stack with count
and optional top-face) and `<Hand>` (horizontal scrolling row of cards).

## Files

- Create `src/ui/deck/DeckStack.tsx`.
- Create `src/ui/hand/Hand.tsx`.
- Create `tests/ui/deck/DeckStack.test.tsx`,
  `tests/ui/hand/Hand.test.tsx`.

## API

```tsx
export interface DeckStackProps<T> {
  cards: ReadonlyArray<T | null>;     // null = redacted (face-down)
  renderTop?: (card: T) => React.ReactNode;
  onClick?: () => void;
  label?: string;                      // e.g. "Battle Deck"
}
export function DeckStack<T>(props: DeckStackProps<T>): JSX.Element;

export interface HandProps<T> {
  cards: ReadonlyArray<T>;
  selectedID?: string;
  onSelect?: (card: T) => void;
  renderCard: (card: T) => React.ReactNode;
}
export function Hand<T extends { id: string } | { name: string }>(
  props: HandProps<T>,
): JSX.Element;
```

## Steps

1. `DeckStack`:
   - Renders `cards.length` as a count badge.
   - If `renderTop && cards[0] !== null`, renders `renderTop(cards[0])` at
     the top of the stack (used when the top has been flipped).
   - Otherwise renders a face-down `<CardFrame>` from 09.2 with the back
     graphic.
   - Click → `onClick` (used to flip).
2. `Hand`:
   - Horizontal scroll, snap to card width.
   - Selected card raised (`elevation` jump).
   - Click → `onSelect`.

## Tests

- `DeckStack` shows count = `cards.length`.
- `DeckStack` with all `null` shows face-down only; with top revealed
  renders `renderTop`.
- `Hand` selected card has higher z-index / elevation.
- Click selects.

## Dependencies

09.2 (CardFrame).

## Out of scope

- Card animations (deal / discard).
