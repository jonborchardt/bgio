# 12.4 — CI workflow

**Parent:** [12-testing-debug.md](12-testing-debug.md)

## Goal

A single GitHub Actions workflow that runs every gate on every PR + push:
typecheck, lint, unit tests, fuzz, smoke e2e, and a build of `dist/`.

## Files

- Create `.github/workflows/ci.yml`.
- Modify `package.json` scripts to ensure each gate has a single npm
  script invocation.

## Steps

1. Workflow triggers: `pull_request`, `push: branches: [main]`.
2. One job (`build-and-test`) running on `ubuntu-latest`, Node 20.
3. Steps:
   ```yaml
   - actions/checkout@v4
   - actions/setup-node@v4 (with cache)
   - run: npm ci
   - run: npm run typecheck
   - run: npm run lint
   - run: npm test                  # unit + fuzz
   - run: npm run build             # ensures vite build works
   - run: npm run e2e:smoke         # added in 12.5
   ```
4. Optional matrix: `numPlayers ∈ {1,2,3,4}` for the fuzz run, but a
   single env var (`FUZZ_PLAYERS=all`) is simpler.
5. Cache: `~/.npm` per `package-lock.json` hash.

## Tests

- A red-PR run blocks merge.
- A green-PR run produces the build artifact.

## Dependencies

01–11 (the gates only matter if we have things to gate).

## Out of scope

- Auto-deploy on green main — that's 13.x.
- Coverage reporting upload to a third party.
