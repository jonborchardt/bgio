# 14.1 — Hot-seat seat picker

**Parent:** [14-playtest-followups.md](14-playtest-followups.md)

## Goal

Make hot-seat playable from a single tab. The current default build
mounts `Client({ game, board, numPlayers: 4 })` with no `playerID`,
so every role panel returns null and the StatusBar reports "Mode:
Spectating" — see [14-playtest-followups.md](14-playtest-followups.md)
issue B1.

This sub-plan adds an in-app seat picker that calls
`client.updatePlayerID(seat)` so each role panel sees the chosen
viewer. bgio's debug panel exposes the same affordance under
`PLAYERS`, but it's dev-only and confusing (its `1` button collides
with the reset shortcut, see V7).

## Files

- Modify `src/Board.tsx` — render `<SeatPicker>` near the top.
- Modify `src/App.tsx` — pass an initial `playerID='0'` to the
  hot-seat `Client` so the chief panel renders by default.
- Create `src/ui/layout/SeatPicker.tsx`.
- Create `tests/ui/layout/SeatPicker.test.tsx` (smoke + it.todo
  until @testing-library/react is wired).

## API

```tsx
export interface SeatPickerProps {
  numPlayers: 1 | 2 | 3 | 4;
  current: PlayerID;
  roleAssignments: Record<PlayerID, Role[]>;
  onChange: (seat: PlayerID) => void;
}

export const SeatPicker: React.FC<SeatPickerProps>;
```

## Steps

1. `<SeatPicker>` renders one MUI Tab per seat, labeled
   `"Player N: <roles>"` (e.g. `"Player 1: chief, science"` for the
   2-player chief seat). Active tab gets `palette.role.<role>` accent.
2. `onChange(seat)` calls `props.client.updatePlayerID(seat)` from
   the parent (`Board.tsx`). bgio re-renders the board with the new
   `playerID`, and each role panel's gating naturally swaps.
3. `Board.tsx` keeps a small `useState<PlayerID>(props.playerID ?? '0')`
   and passes it to the picker. The bgio Client is the source of
   truth for the actual seat; we just mirror it for the picker UI.
4. In hot-seat, `App.tsx` mounts `Client({ ..., playerID: '0' })` so
   the first render shows the chief panel rather than spectator
   mode. The picker then lets the user flip to any other seat.
5. In networked mode the seat is fixed by the lobby join — render the
   picker as a read-only "You are Player N" badge instead of a tab
   row.

## Tests

- 4-player hot-seat boots with chief panel visible (regression for
  the playtest finding).
- Clicking the "Player 2" tab calls `client.updatePlayerID('1')` and
  the SciencePanel renders.
- 2-player game: tab labels show the doubled-up roles
  ("Player 1: chief, science").
- Networked mode renders the read-only badge, not the tab row.

## Dependencies

01.1 (assignRoles), 04.5/05.5/06.7/07.7 (the panels themselves).

## Out of scope

- Persisting the picker's last-active seat across reloads. Hot-seat
  is ephemeral.
- Per-seat keyboard shortcuts.
