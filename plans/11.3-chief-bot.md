# 11.3 — ChiefBot

**Parent:** [11-ai.md](11-ai.md)

## Goal

A heuristic bot for the Chief role that distributes resources proportional
to demand: Foreign upkeep, Domestic in-flight purchases, and Science
contributions in-progress.

## Files

- Create `src/game/ai/chiefBot.ts`.
- Create `tests/ai/chiefBot.test.ts`.

## API

```ts
import type { BotAction } from 'boardgame.io/ai';

export const chiefBot: {
  play: (state: { G: SettlementState; ctx: Ctx }) => BotAction;
};
```

## Steps

1. Read demand:
   - `foreignDemand`: total upkeep cost for `inPlay` units (07.2 helpers).
   - `domesticDemand`: cheapest unbought hand card's cost.
   - `scienceDemand`: smallest "remaining cost" across non-completed cards.
2. Allocate the bank in proportion to demand (rounded down).
3. Issue one `chiefDistribute` per non-chief seat that has demand.
4. End with `chiefEndPhase`.
5. Edge: if bank is empty, just end the phase.

## Tests

- 4-player game, all roles distinct: bot distributes proportionally; each
  seat ends up with at least 1 gold if total demand fits.
- 1-player game: no distribution (no other seats); bot just ends phase.
- Empty bank: bot ends phase.
- Heuristic determinism: same state → same allocation.

## Dependencies

04, 11.2 (so the bot can register via the same hooks).
