# 12.1 — Test helper library

**Parent:** [12-testing-debug.md](12-testing-debug.md)

## Goal

A shared, typed test helper library so every later test file looks the
same. Generalizes 01.4's `makeClient` and adds factories for common
mid-game states.

## Files

- Modify `tests/helpers/makeClient.ts` (already created in 01.4).
- Create `tests/helpers/factories.ts` — mid-game state builders.
- Create `tests/helpers/assertConservation.ts`.
- Create `tests/helpers/seeds.ts` — named seed constants.

## API

```ts
// tests/helpers/factories.ts
export const seedFreshGame: (numPlayers?: 1|2|3|4) => SettlementState;
export const seedAfterChiefDistribute: (
  partial?: { [seat: string]: Partial<ResourceBag> },
) => SettlementState;
export const seedWithBuilding: (defID: string, x: number, y: number) => SettlementState;
export const seedWithUnit: (defID: string, count?: number) => SettlementState;
export const seedMidScienceProgress: (cardID: string, paid: Partial<ResourceBag>) => SettlementState;

// tests/helpers/seeds.ts
export const SEEDS = {
  STABLE: 'test-seed-stable',
  RECRUIT_HEAVY: 'test-seed-recruits',
  ECONOMY: 'test-seed-economy',
} as const;
```

## Steps

1. Each factory builds a `SettlementState` directly (bypassing setup) so
   tests don't have to play a long opening to reach a state of interest.
2. Factories compose: `seedAfterChiefDistribute` calls `seedFreshGame`
   first, then mutates.
3. `assertConservation` is exported from 11.1's invariants for reuse here.

## Tests

- Each factory output passes `assertNoNegativeResources`.
- `seedFreshGame(4)` matches `Settlement.setup` output for 4 players bit
  for bit.

## Dependencies

01.4, 11.1.

## Out of scope

- Snapshot testing of seeds. Tests should verify behavior, not exact bytes
  of seeds (which change as content lands).
