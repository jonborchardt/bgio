# 05.2 — scienceContribute move

**Parent:** [05-science.md](05-science.md)

## Goal

Allow Science to incrementally pay resources toward a science card's cost,
respecting the "may only advance the lowest number card in each color" rule.

## Files

- Create `src/game/roles/science/contribute.ts`.
- Register move in `src/game/moves.ts`.
- Create `tests/roles/science/contribute.test.ts`.

## API

```ts
export const scienceContribute: Move<SettlementState>;
// args: cardID: string, amounts: Partial<ResourceBag>
```

## Steps

1. Validate caller seat holds `science` and is in stage `scienceTurn`.
2. Look up `card = grid.find(c => c.id === cardID)`.
3. Validate card is on the grid and not yet completed.
4. **Lowest-first rule:** find the lowest-level non-completed card in the
   same color column; if it isn't `card`, return `INVALID_MOVE`.
5. Validate `canAfford(G.wallets[playerID], amounts)`.
6. Move resources: `transfer(G.wallets[playerID], G.science.paid[cardID],
   amounts)`. Cap each resource at the card's `cost` (excess returns to
   wallet — caller error, but be defensive).

## Tests

- Pay 2 gold of a 5 gold card → `paid` has 2 gold; wallet down 2 gold.
- Pay more than wallet has → `INVALID_MOVE`, state unchanged.
- Pay toward a higher-level card while a lower-level same-color card is
  unfinished → `INVALID_MOVE`.
- Pay toward the only remaining color card after lower ones complete →
  succeeds.
- Overpay (amount > remaining cost): excess returns to wallet, paid bag
  matches cost exactly.
