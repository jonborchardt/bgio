# 02.1 — Phase skeleton: chiefPhase / othersPhase / endOfRound

**Parent:** [02-engine.md](02-engine.md)

## Goal

Define the three phases that structure a round, with empty hook bodies, so
later stages can wire move sets and round-end behaviors into well-known
extension points.

## Files

- Create `src/game/phases/index.ts` — barrel.
- Create `src/game/phases/chief.ts`, `phases/others.ts`, `phases/endOfRound.ts`.
- Modify `src/game/index.ts` to register `phases` on the `Settlement` Game.
- Create `tests/phases.test.ts`.

## API

```ts
import type { PhaseConfig } from 'boardgame.io';
import type { SettlementState } from '../types';

export const chiefPhase: PhaseConfig<SettlementState>;
export const othersPhase: PhaseConfig<SettlementState>;
export const endOfRound: PhaseConfig<SettlementState>;
```

## Steps

1. `chiefPhase` config:
   - `start: true`
   - `next: 'othersPhase'`
   - `turn.activePlayers`: only the seat holding the `chief` role (resolved
     from `G.roleAssignments` in `onBegin`).
   - `endIf`: returns true when an internal `phaseDone` flag is set; the
     flag is flipped by 04.2's `chiefEndPhase` move (added later).
2. `othersPhase` config:
   - `next: 'endOfRound'`
   - `turn.activePlayers`: every non-chief seat in a placeholder
     "waiting" stage; 02.2 fleshes out the stage map.
   - `endIf`: every non-chief seat has set their per-player `done` flag.
3. `endOfRound` config:
   - `next: 'chiefPhase'`
   - `onBegin`: invokes the round-end hook list from 02.5 (until 02.5
     lands, the function is an empty array).
   - `onEnd`: increments `G.round`.
4. Wire all three into the `Settlement` game's `phases` field.

## Tests

- A fresh client lands in `chiefPhase`.
- Manually flipping the chief's `phaseDone` advances to `othersPhase`.
- Setting all non-chief `done` flags advances to `endOfRound`.
- After `endOfRound` the game returns to `chiefPhase` with `G.round + 1`.

## Dependencies

01.1, 01.3.

## Out of scope

- The `phaseDone` / `done` flag plumbing belongs to 02.2 (active players)
  and the role stubs that own each phase. Here we only define the phase
  shape.
