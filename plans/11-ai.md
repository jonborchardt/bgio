## Goal

Bots good enough to (a) fill empty seats so 1-player solo works without a
human juggling four roles, and (b) act as a regression harness — random bots
playing thousands of games surface engine bugs cheaply.

## Scope

**In:**
- bgio `RandomBot` integration — drives any role with uniformly random
  legal moves. Used for fuzz testing.
- `MCTSBot` integration — used as a baseline for the Chief and as a sanity
  test that the move surface is enumerable.
- **Bot determinism:** all bots derive their tie-breaking randomness from
  bgio's seeded PRNG (02.3). Same seed + same game state = same move
  sequence. This is what tests + replay rely on. Live multiplayer games
  use a server-generated seed so play feels distinct between sessions.
- **Bots run server-side, always.** Solo, idle-takeover (10.9), and any
  future "fill empty seat" feature share one bot infrastructure inside
  the bgio server. Client never instantiates a bot. The dev path is the
  same: `npm run server:dev` boots the server with bots; the client just
  sees a normally-driven match.
- Per-role heuristic bots:
  - `ChiefBot` — distribute proportional to each player's outstanding
    obligations (Foreign upkeep, Domestic in-flight purchases, Science
    contributions).
  - `ScienceBot` — greedy lowest-cost completion.
  - `DomesticBot` — buy any affordable building, prefer adjacency bonuses
    when 06's adjacency content lands.
  - `ForeignBot` — recruit until bank empty, flip until lethal damage
    isn't survivable.
- Solo mode: a single human player owns one role; the rest are bots. Wire
  via bgio's `MatchData.bots` config.

**Out:**
- A "good" general-purpose bot. We optimize for "doesn't crash and finishes
  a game" first.
- Learning / RL.

## Test surface

- 100-game RandomBot fuzz passes with zero invariant violations.
- Each heuristic bot finishes a 60-turn game without raising
  `INVALID_MOVE`.
- Solo mode renders the human's role and reads-only the bot roles.
