# 10.6 — Reconnect & credentials

**Parent:** [10-multiplayer.md](10-multiplayer.md)

## Goal

A page reload mid-game restores the player's seat and credentials. A short
disconnect (network blip) doesn't lose the player's place.

## Files

- Create `src/lobby/credentials.ts` — `localStorage` helpers.
- Modify `src/App.tsx` to read credentials at mount and (if present)
  bypass the lobby.
- Modify `src/lobby/LobbyApp.tsx` to write credentials on join.
- Create `tests/lobby/credentials.test.ts`.

## API

```ts
export interface SessionCreds {
  matchID: string;
  playerID: string;
  credentials: string;
  serverUrl: string;
  expiresAt: number;
}

export const saveCreds: (creds: SessionCreds) => void;
export const loadCreds: () => SessionCreds | null;
export const clearCreds: () => void;
```

## Steps

1. Save under a single localStorage key (`settlement.session`) with a
   24-hour TTL (game-design.md target 60 turns; 24h is generous).
2. On mount: `loadCreds()`. If present and the server responds 200 to a
   "still in match?" probe, mount the game `Client` directly. Else
   `clearCreds()` and route to the lobby.
3. The probe is a small fetch to `/games/settlement/<matchID>` checking
   the player is still listed.
4. SocketIO transport handles reconnection automatically; we just need to
   ensure it has the right credentials at construct time.
5. **Server-down behavior:** if the probe fails or the SocketIO connect
   throws, render a `<ServerDownSpinner>` with retry-with-backoff
   (1s, 2s, 5s, 15s, 30s, 60s, capped) and a manual "retry now" button.
   No auto-fallback to hot-seat — fall-back would silently diverge from
   server state and confuse the player on reconnect. `<ServerDownSpinner>`
   stays visible for the lifetime of the outage.

## Tests

- `saveCreds` then `loadCreds` round-trips.
- Expired creds → `loadCreds` returns `null`.
- `App` skips lobby when valid creds exist.
- Server-down spinner renders when probe fails; retry backoff fires at
  the documented intervals; manual retry resets the backoff.

## Dependencies

10.2, 10.3.

## Out of scope

- "Kick if absent for N rounds" — server-side concern, post-V1.
