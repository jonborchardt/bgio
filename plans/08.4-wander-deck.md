# 08.4 — Wander deck (opponent, Option 1)

**Parent:** [08-cross-cutting.md](08-cross-cutting.md)

## Goal

Implement the wander-deck opponent: at each round end, draw one wander card
that bonuses or hurts our civ. Per game-design.md §Card Counts: 24 wander
cards (8x3) + 8 directional.

## Files

- Create `src/data/wanderCards.json`.
- Create `src/data/wanderCards.ts` — typed loader.
- Create `src/game/opponent/wanderDeck.ts`.
- Modify `src/game/setup.ts` to call `setupWanderDeck`.
- Create `tests/opponent/wanderDeck.test.ts`.

## API

```ts
export interface WanderCardDef {
  id: string;
  name: string;
  effects: EventEffect[];   // reuses dispatcher from 08.2
  flavor?: string;
}

export interface WanderState {
  deck: WanderCardDef[];     // top of deck = next round's draw
  discard: WanderCardDef[];
  currentlyApplied: WanderCardDef | null;
}

export const setupWanderDeck: (random: RandomAPI) => WanderState;
```

## Steps

1. Loader validates 24 + 8 = 32 cards (or whatever count we ship).
2. `setupWanderDeck` shuffles all wander cards into `deck`; `discard = []`.
3. Register a round-end hook `opponent:wander-step` (via 02.5) that:
   - Pushes the previous `currentlyApplied` (if any) to `discard` and
     reverses any effects flagged "duration: oneRound".
   - Pops the top of `deck` into `currentlyApplied`.
   - Calls `dispatch(...)` from 08.2 for each effect.
   - When the deck empties, reshuffle `discard` into `deck`.
4. `playerView`: deck order is hidden from all players (everyone's a
   player here). Top card is *revealed* once applied, so it lands in
   `currentlyApplied` visibly.

## Tests

- 32 cards in deck after setup.
- Round-end hook applies one card and increments discard count.
- Deck reshuffle when empty.
- Hook order: previous card's transient effects unwound before new card
  applies.
- `playerView` redacts deck order but exposes `currentlyApplied`.
