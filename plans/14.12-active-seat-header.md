# 14.12 — Active-seat board header

**Parent:** [14-playtest-followups.md](14-playtest-followups.md)

## Goal

Fix the misleading "Player N's turn" header at the top of the
`SettlementBoard` so it reflects the **active** seat (the one bgio's
engine will accept moves from), not just `ctx.currentPlayer`.

## Why

Driven by a 2026-04-29 hot-seat smoke (post-14.1 / 14.2): after the
foreign seat ends round 0, the game transitions back to `chiefPhase`
for round 1. The chief seat (Player 1) is the only seat bgio will
accept moves from in `chiefPhase`, but the header reads
**"Player 4's turn"** because `ctx.currentPlayer` still points at the
seat that last moved in `othersPhase`. A new player reading the
header has no idea who's actually up.

bgio uses two cooperating concepts:

- `ctx.currentPlayer` — the "default" seat; advances after every
  `pass`-style turn end. In phases with `setActivePlayers`, it can
  trail the actual active seat by an entire phase.
- `ctx.activePlayers` — the per-seat stage map. When set, *this* is
  the source of truth for who can move.

When `activePlayers` is set, the header should pick a seat from it.
When it's null (e.g. fall-through phases), `currentPlayer` stays
correct.

## Files

- Modify `src/Board.tsx` — replace the inline `ctx.currentPlayer`
  read with a small helper.
- Create `src/ui/layout/activeSeat.ts` — pure helper that picks the
  active seat from `(ctx.activePlayers, ctx.currentPlayer,
  G.roleAssignments)` and returns both the seat id and a label
  (`"Player 1: chief"`).
- Create `tests/ui/layout/activeSeat.test.ts` — table-driven cases
  for chiefPhase, othersPhase mid-round, gameover.

## API

```ts
export interface ActiveSeatInfo {
  seat: PlayerID;       // the seat id bgio will accept moves from
  label: string;        // "Player N: <roles>"
  /** True when the local seat matches the active seat — Board uses this
   *  to subtly highlight or de-emphasize the header. */
  isLocal: boolean;
}

export function pickActiveSeat(args: {
  activePlayers: Record<PlayerID, string> | null | undefined;
  currentPlayer: PlayerID;
  roleAssignments: Record<PlayerID, Role[]>;
  localSeat: PlayerID | null | undefined;
}): ActiveSeatInfo;
```

## Steps

1. **Helper.** When `activePlayers` is set and non-empty, pick a seat
   whose value is *not* `'done'` (those seats have already finished
   their work this phase). If multiple seats remain, pick the lowest
   seat id deterministically — the helper has no notion of priority,
   it only needs to be stable. When `activePlayers` is empty / null,
   fall back to `currentPlayer`.
2. **Label.** Build `"Player ${seatNum}: <roles>"` from
   `roleAssignments[seat]`. Mirror the SeatPicker's label format
   exactly so the header and the picker tab agree.
3. **Board.** Replace
   ``` `Player ${Number(ctx.currentPlayer) + 1}'s turn` ```
   with the helper's label. Keep the existing `gameOver` branch.
   Drop the dead `Number(ctx.currentPlayer) + 1` arithmetic.
4. **Test.** Cover (a) chiefPhase 4p where activePlayers maps only
   the chief seat, (b) othersPhase mid-round with the chief in
   `done` and 2 of 3 non-chief seats still pending, (c) gameover
   path (Board branches before calling the helper, so this just
   asserts the helper still returns *something* for safety), (d)
   no-active-players fallback to `currentPlayer`.

## Tests

- `pickActiveSeat` returns the chief seat in 4p chiefPhase.
- Returns the lowest pending non-chief seat in othersPhase when one
  has already flipped done.
- Falls back to `currentPlayer` when `activePlayers` is null.
- Returns the only non-`done` seat when activePlayers has the chief
  in `done` and a single non-chief seat pending.
