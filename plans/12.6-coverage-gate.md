# 12.6 — Coverage gate

**Parent:** [12-testing-debug.md](12-testing-debug.md)

## Goal

Run `vitest --coverage` with thresholds that gate pure-logic modules at
≥90% but don't penalize UI files (which are tested behaviorally rather
than line-by-line).

## Files

- Modify [vite.config.ts](../vite.config.ts) — vitest `coverage` block.
- Modify `package.json`: `test:coverage`.
- Modify CI workflow (12.4) to run `npm run test:coverage` instead of
  `npm test`.
- Create `tests/coverage.note.md` — short doc explaining the policy.

## Steps

1. Add `@vitest/coverage-v8` devDependency.
2. Vitest config:
   ```ts
   coverage: {
     provider: 'v8',
     reporter: ['text', 'lcov'],
     include: ['src/**/*.{ts,tsx}'],
     exclude: ['src/**/*.tsx', 'src/main.tsx', 'src/theme.ts'],
     thresholds: { lines: 90, functions: 90, branches: 80 },
   }
   ```
3. The exclusion of `*.tsx` is intentional: behavioral tests cover them.
   We can revisit if a complex non-render hook lives in a `.tsx` file.
4. CI fails when thresholds drop.

## Tests

- `npm run test:coverage` produces `coverage/lcov.info` and prints a table.
- Lowering coverage of a covered module makes CI red.
