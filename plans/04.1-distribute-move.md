# 04.1 — chiefDistribute move

**Parent:** [04-chief.md](04-chief.md)

## Goal

The Chief takes resources from the bank and places them into the chosen
non-chief player's circle on the center mat (game-design.md §General Play
Chief).

## Files

- Create `src/game/roles/chief/distribute.ts`.
- Register the move in `src/game/moves.ts`.
- Create `tests/roles/chief/distribute.test.ts`.

## API

```ts
import type { Move } from 'boardgame.io';
import type { PlayerID, ResourceBag } from '../../types';

export const chiefDistribute: Move<SettlementState>;
// args: targetSeat: PlayerID, amounts: Partial<ResourceBag>
```

## Steps

1. Validate caller seat holds the `chief` role
   (`rolesAtSeat(G.roleAssignments, ctx.playerID).includes('chief')`).
2. Validate `ctx.phase === 'chiefPhase'`.
3. Validate `targetSeat` is non-chief and has a circle on the mat.
4. Validate `canAfford(G.bank, amounts)`.
5. Use `transfer(G.bank, G.centerMat.circles[targetSeat], amounts)`.
6. Any failure → `INVALID_MOVE`. State unchanged on failure (Immer rollback
   via early-return before mutation).

## Tests

- Happy path: 2-player game, chief moves 2 gold to seat 1; bank loses 2 gold,
  seat 1 circle gains 2 gold.
- Wrong phase: `chiefDistribute` during `othersPhase` returns
  `INVALID_MOVE`.
- Non-chief caller: returns `INVALID_MOVE`.
- Underflow: bank has 1 gold, distribute 2 gold returns `INVALID_MOVE` and
  bank still has 1 gold (state unchanged).
- Self-target (chief tries to distribute to own seat): returns
  `INVALID_MOVE`.

## Dependencies

01.1, 02.1, 03.1, 03.2, 03.3.

## Out of scope

- The chief ending their phase (04.2).
- Multi-target single-call distribution: callers issue one move per target
  to keep the move surface small.
