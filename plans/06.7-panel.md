# 06.7 — Domestic UI panel

**Parent:** [06-domestic.md](06-domestic.md)

## Goal

A two-region panel: hand row at the top, in-play grid below. Drag-or-click
to place. Cells highlight legality on hover.

## Files

- Create `src/ui/domestic/DomesticPanel.tsx`.
- Create `src/ui/domestic/Hand.tsx`.
- Create `src/ui/domestic/BuildingGrid.tsx`.
- Create `src/ui/domestic/CellSlot.tsx`.
- Modify `src/Board.tsx` to render `<DomesticPanel>` for the seat holding
  Domestic.
- Create `tests/ui/domestic/DomesticPanel.test.tsx`.

## API

```tsx
export const DomesticPanel: React.FC<BoardProps<SettlementState>>;
```

## Steps

1. `<Hand>` lists `<BuildingCard>`s (from 09.2). Selecting one enters a
   "placing" mode held in panel-local state.
2. `<BuildingGrid>` is a CSS grid with cells from `min(x)-1` to `max(x)+1`
   (i.e. one ring around current placements). Each `<CellSlot>` knows
   whether it's legal for the active card.
3. Click on a legal cell → `props.moves.domesticBuyBuilding(cardName, x, y)`.
4. In-play building click → opens an upgrade affordance →
   `domesticUpgradeBuilding(x,y, ...)`.
5. "Produce" button at bottom → `domesticProduce()` (greyed if already
   produced this round).

## Tests

- Empty grid: clicking the center cell while a card is active fires
  `domesticBuyBuilding`.
- Once a building exists, only neighbors highlight as legal.
- "Produce" button disables after one click in a round.

## Dependencies

06.1, 06.2, 06.4, 09.2 (BuildingCard component). Soft dep on 09.4 theme.

## Out of scope

- Drag-and-drop. Click-to-place is enough for V1.
- Worker placement UI (chief acts here — 04.3 wires the click handler;
  this panel renders a worker indicator on the cell).
