# 11.6 — ForeignBot

**Parent:** [11-ai.md](11-ai.md)

## Goal

Heuristic for Foreign: pay upkeep, recruit until wallet empty, flip battles
while the resolver predicts a survivable win.

## Files

- Create `src/game/ai/foreignBot.ts`.
- Create `tests/ai/foreignBot.test.ts`.

## API

```ts
export const foreignBot: { play: (state) => BotAction };
```

## Steps

1. **Upkeep:** call `foreignUpkeep`. If insufficient wallet, release the
   cheapest unit until upkeep fits, then upkeep.
2. **Recruit loop:** while wallet covers the cheapest hand unit, recruit
   it (cheapest first). Stop when wallet can't afford anything.
3. **Flip loop:**
   - Peek at `battleDeck[0]` (bot has access — same seat).
   - Call resolver in a "what-if" mode with a greedy damage allocation
     (lowest-HP unit absorbs first).
   - If the predicted outcome is `win` AND post-battle player roster has
     ≥50% HP remaining: `foreignFlipBattle`, then `foreignAssignDamage`
     with the same plan.
   - If the resolver predicts `lose` or marginal: stop flipping.
4. End phase.

## Tests

- Bot releases a unit when wallet < upkeep.
- Recruits in cost order until wallet drained.
- Flips one battle when resolver predicts a confident win.
- Stops flipping when prediction is `'lose'`.

## Dependencies

07, 11.2.

## Out of scope

- Optimal damage allocation. Greedy "lowest HP first" is simple and
  reasonable.
- Trade strategy (we always flip trades after a win in V1).
