# 06.1 — Domestic hand & in-play grid

**Parent:** [06-domestic.md](06-domestic.md)

## Goal

State + placement validation for the Domestic player's hand and in-play
building grid (Option 1, adjacency-required).

## Files

- Create `src/game/roles/domestic/types.ts`.
- Create `src/game/roles/domestic/grid.ts` — pure helpers.
- Modify `src/game/setup.ts` to call `setupDomestic`.
- Create `tests/roles/domestic/grid.test.ts`.

## API

```ts
export interface DomesticBuilding {
  defID: string;     // matches BuildingDef.name
  upgrades: number;  // 0 by default
  worker: { ownerSeat: PlayerID } | null;
}

export interface DomesticState {
  hand: BuildingDef[];
  grid: Map<string, DomesticBuilding>; // key: 'x,y'
}

export const cellKey = (x: number, y: number): string;
export const isOrthogonallyAdjacent = (a: string, b: string): boolean;
export const isPlacementLegal = (
  grid: Map<string, DomesticBuilding>, x: number, y: number,
): boolean;
export const setupDomestic = (
  techsAlreadyUsedBy: Set<string>,
): DomesticState;
```

## Steps

1. `cellKey(x,y) = \`${x},${y}\``.
2. `isOrthogonallyAdjacent` — Manhattan distance == 1.
3. `isPlacementLegal`:
   - Cell must be empty.
   - If grid is empty, any cell is legal.
   - Otherwise, must be adjacent to some existing cell.
4. `setupDomestic`:
   - Hand starts with all level-0 tech cards from `TECHNOLOGIES` whose
     branch corresponds to Domestic and that aren't in
     `techsAlreadyUsedBy` (set populated by 05.1's `underCards`).
   - Grid starts empty.

## Tests

- First placement at any cell is legal; subsequent placements require
  adjacency.
- `cellKey` is stable.
- Placement fails on occupied cell.
- `setupDomestic` excludes any tech in the science under-cards set.

## Dependencies

01.2, 02.5, 05.1 (so we know which techs are unavailable). 03 (resources)
soft-dep for cost types.
