# 10.7 — Accounts & run history

**Parent:** [10-multiplayer.md](10-multiplayer.md)

## Goal

Username/password accounts on the server, plus a `runs` table keyed to
the user that records every game's outcome (win-turns or time-up score).
Lobby joins are gated on a logged-in user.

## Files

- Create `server/auth/accounts.ts` — register / login / verify.
- Create `server/auth/passwordHash.ts` — argon2id wrapper.
- Create `server/auth/middleware.ts` — Koa middleware that resolves a
  user from cookie / Authorization header.
- Create `server/runs/runs.ts` — record + query run history.
- Modify `server/storage/sqlite.ts` (and migrations) to add `users`
  and `runs` tables.
- Create `src/lobby/AuthForms.tsx` — login + register UI.
- Modify `src/lobby/LobbyApp.tsx` to gate the lobby on a logged-in user.
- Create `src/lobby/authClient.ts` — fetch wrappers for auth endpoints.
- Create `tests/server/auth.test.ts`, `tests/server/runs.test.ts`.

## API

```ts
// server/auth/accounts.ts
export interface User {
  id: string;          // uuid
  username: string;    // unique, case-insensitive
  createdAt: number;
}

export const register: (username: string, password: string) => Promise<User>;
export const login:    (username: string, password: string) => Promise<{ user: User; token: string }>;
export const verify:   (token: string) => Promise<User | null>;

// server/runs/runs.ts
export interface RunRecord {
  id: string;
  userID: string;
  matchID: string;
  outcome: 'win' | 'timeUp';
  turns: number;
  settlementsJoined: number;
  createdAt: number;
}

export const recordRun: (rec: Omit<RunRecord, 'id' | 'createdAt'>) => Promise<RunRecord>;
export const listRunsByUser: (userID: string) => Promise<RunRecord[]>;
export const personalBest: (userID: string) => Promise<{
  fastestWinTurns: number | null;
  bestTimeUpSettlements: number | null;
}>;
```

## Validation rules

- **Username:** 3–20 chars, case-insensitive unique, allowed
  `[A-Za-z0-9_-]`. Trim on submit.
- **Password:** min 8 chars, no max, no complexity requirements. Hashed
  with argon2id; never stored or logged in plaintext.

## Steps

1. **Schema** (added to migrations under [13.3](13.3-database-choice.md)):
   ```sql
   CREATE TABLE users (
     id TEXT PRIMARY KEY,
     username TEXT UNIQUE NOT NULL COLLATE NOCASE,
     password_hash TEXT NOT NULL,
     created_at INTEGER NOT NULL
   );
   CREATE TABLE auth_tokens (
     token TEXT PRIMARY KEY,
     user_id TEXT NOT NULL REFERENCES users(id),
     expires_at INTEGER NOT NULL
   );
   CREATE TABLE runs (
     id TEXT PRIMARY KEY,
     user_id TEXT NOT NULL REFERENCES users(id),
     match_id TEXT NOT NULL,
     outcome TEXT NOT NULL,            -- 'win' | 'timeUp'
     turns INTEGER NOT NULL,
     settlements_joined INTEGER NOT NULL,
     created_at INTEGER NOT NULL
   );
   CREATE INDEX runs_by_user ON runs(user_id, created_at DESC);
   ```
2. Password hashing via `argon2id`. Token = random 32-byte hex; 24h
   expiry; rotate on `verify` if older than 1h.
3. Lobby gate: until `verify` succeeds, the lobby renders `<AuthForms>`
   (login + register tabs).
4. `joinMatch` (10.3) carries the token; the server middleware resolves
   the user and binds it to the seat metadata.
5. **Run persistence hook:** server-side bgio plugin listens for the
   `endIf` outcome (08.5). When fired, calls `recordRun` once per
   distinct user-seat in the match. Idempotent on `(matchID, userID)`.
6. `personalBest` powers the win screen line "fastest win: 42 turns —
   try to beat it".

## Tests

- Register / login round-trip; wrong password fails.
- Token verify works for an hour; auto-rotates near expiry.
- `recordRun` is idempotent per `(matchID, userID)`.
- `personalBest` returns the lowest `turns` across `outcome='win'` and
  the highest `settlementsJoined` across `outcome='timeUp'`.
- Lobby is inaccessible without a valid token.

## Dependencies

## Out of scope

- OAuth / SSO (Google, GitHub). Add later if there's demand.
- Password reset email flow. V1 ships with a "remember your password"
  warning. (Acceptable for a hobby project; revisit if it stings.)
- Public leaderboards across users.
