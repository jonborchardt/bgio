# 05.4 — sciencePlayBlueEvent (stub)

**Parent:** [05-science.md](05-science.md)

## Goal

Declare the move surface for the Science player's blue event card. Mirrors
04.4 for the Chief.

## Files

- Create `src/game/roles/science/playBlueEvent.ts`.
- Register in `src/game/moves.ts`.
- Create `tests/roles/science/playBlueEvent.test.ts`.

## API

```ts
export const sciencePlayBlueEvent: Move<SettlementState>;
// args: cardID: string
```

## Steps

1. Validate caller seat holds `science`.
2. Validate it is the science player's stage *or* an interrupt is allowed
   (events.md says "may play 1 ... at any time" — once we have the stage
   stack from 02.2, allow this from any of the seat's owned stages).
3. Validate card is in `G.events.hands.blue[seat]` and unused this cycle.
4. Dispatch via 08.2's dispatcher.
5. Mark used; set `G._eventPlayedThisRound[seat] = 'blue'`.

Until 08 lands: short-circuit `INVALID_MOVE` when `G.events` is undefined.

## Tests

- Without 08: `INVALID_MOVE`.
- With stub: card consumed; dispatcher called.
- Playing two blue events in one round → second is `INVALID_MOVE`.
- Wrong color in hand → `INVALID_MOVE`.

## Dependencies

01.1, 02.1, 02.2.
