# 09.2 — Reusable card components

**Parent:** [09-ui.md](09-ui.md)

## Goal

A typed family of card components, each driven from JSON data via 01.2's
loaders. Zero hardcoded card text in components.

## Files

- Create `src/ui/cards/CardFrame.tsx` — visual chrome.
- Create `src/ui/cards/BuildingCard.tsx`.
- Create `src/ui/cards/UnitCard.tsx`.
- Create `src/ui/cards/ScienceCard.tsx`.
- Create `src/ui/cards/TechCard.tsx`.
- Create `src/ui/cards/EventCard.tsx`.
- Create `tests/ui/cards/cards.test.tsx`.

## API

```tsx
export interface CardFrameProps {
  tier?: 'beginner' | 'intermediate' | 'advanced';
  color?: 'red' | 'gold' | 'green' | 'blue';
  children: React.ReactNode;
}
export const CardFrame: React.FC<CardFrameProps>;

export const BuildingCard: React.FC<{ def: BuildingDef; count?: number }>;
export const UnitCard: React.FC<{ def: UnitDef; count?: number }>;
export const ScienceCard: React.FC<{ def: ScienceCardDef; paid: ResourceBag }>;
export const TechCard: React.FC<{ def: TechnologyDef }>;
export const EventCard: React.FC<{ def: EventCardDef; faceDown?: boolean }>;
```

## Steps

1. `CardFrame` draws a MUI `Paper` with theme-driven `tier` border + `color`
   accent (tokens added in 09.4).
2. Each typed card renders fields via `Typography` + `Stack` from MUI.
   Resource fields use `<ResourceBag>` from 09.3.
3. `count?: number` renders a "×N" badge — used for stacked unit instances
   in the Foreign army row.
4. `faceDown` on `EventCard` shows the card back only.

## Tests

- `BuildingCard` renders `def.name`, `def.cost`, and `def.benefit` strings
  exactly as in JSON.
- `UnitCard count={3}` shows "×3".
- `EventCard faceDown` does NOT render the card name.
- Snapshot of one of each card type for regression detection (Vitest
  inline snapshots).
