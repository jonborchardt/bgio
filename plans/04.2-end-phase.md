# 04.2 — chiefEndPhase move

**Parent:** [04-chief.md](04-chief.md)

## Goal

Chief explicitly ends their phase. Engine transitions to `othersPhase` with
all non-chief seats activated in their respective stages.

## Files

- Create `src/game/roles/chief/endPhase.ts`.
- Register move in `src/game/moves.ts`.
- Modify `src/game/phases/chief.ts` to gate `endIf` on the flag this move
  sets.
- Create `tests/roles/chief/endPhase.test.ts`.

## API

```ts
export const chiefEndPhase: Move<SettlementState>;
// no args
```

## Steps

1. Validate caller holds chief and `ctx.phase === 'chiefPhase'`.
2. Set `G._phaseDone.chief = true`. (Internal flag namespaced under
   `_phaseDone`; underscores indicate engine-internal.)
3. `chiefPhase.endIf` checks `G._phaseDone.chief`. When true, bgio transitions
   to `othersPhase`.
4. `othersPhase.turn.onBegin` calls
   `setActivePlayers({ value: activePlayersForOthers(G.roleAssignments) })`
   from 02.2.

## Tests

- Calling `chiefEndPhase` lands the game in `othersPhase`.
- Active players in `othersPhase`: every non-chief seat is in its expected
  stage; chief seat is in `done`.
- Calling `chiefEndPhase` outside `chiefPhase` returns `INVALID_MOVE`.
- Calling twice in the same phase is a no-op (already true).

## Dependencies

02.1, 02.2, 04.1 (so the test can verify the chief actually distributed
before ending — not strictly required, but the test covers the realistic
flow).

## Out of scope

- Worker placement (04.3).
- Gold event (04.4).
