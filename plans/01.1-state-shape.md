# 01.1 — State shape & role assignment

**Parent:** [01-foundations.md](01-foundations.md)

## Goal

Define the canonical `SettlementState` plus the deterministic mapping from
`numPlayers` to which seat plays which roles, per game-design.md §Players.

## Files

- Create `src/game/types.ts` — exported types only, zero runtime.
- Create `src/game/roles.ts` — pure helpers, no bgio imports.
- Create `tests/roles.test.ts`.

## API

```ts
// src/game/types.ts
export type Role = 'chief' | 'science' | 'domestic' | 'foreign';
export type PlayerID = string;             // bgio uses '0', '1', ...

export interface SettlementState {
  // public
  bank: ResourceBag;                       // imported from 03.1; for now: Record<string, number>
  centerMat: CenterMat;                    // 03.3 fills this; placeholder shape OK
  roleAssignments: Record<PlayerID, Role[]>;
  round: number;
  // private slices populated by 02.4
  hands: Record<PlayerID, unknown>;        // refined per role
  // decks belong to whoever owns them; placed under players who hold them
}

// src/game/roles.ts
export const assignRoles = (numPlayers: 1 | 2 | 3 | 4): Record<PlayerID, Role[]>;
export const seatOfRole = (assignments: Record<PlayerID, Role[]>, role: Role): PlayerID;
export const rolesAtSeat = (assignments: Record<PlayerID, Role[]>, seat: PlayerID): Role[];
```

## Steps

1. Write the role-assignment table verbatim from game-design.md:
   - 1p: `{ '0': ['chief','science','domestic','foreign'] }`
   - 2p: `{ '0': ['chief','science'], '1': ['domestic','foreign'] }`
   - 3p: `{ '0': ['chief','science'], '1': ['domestic'], '2': ['foreign'] }`
   - 4p: one role per seat.
2. `assignRoles` is a pure lookup; throw on out-of-range.
3. `seatOfRole` finds the unique seat for a role; throw on duplicates.
4. `rolesAtSeat` returns the array (or `[]` if seat not present).
5. Export everything from `src/game/index.ts` barrel (created here even
   though it only has these two modules so far).

## Tests

- `assignRoles(2)` matches the literal table above.
- `seatOfRole(assignRoles(3), 'foreign')` returns `'2'`.
- `rolesAtSeat(assignRoles(1), '0')` returns all four roles.
- `assignRoles(5 as any)` throws.

## Dependencies

None.

## Out of scope

- The actual content of `bank`, `centerMat`, `hands` — placeholders only.
  Stage 03 fills these; per-role hand types come from each role stub.
