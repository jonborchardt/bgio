# 03.1 — Token types & ResourceBag

**Parent:** [03-resources.md](03-resources.md)

## Goal

A small, exhaustive set of resource tokens plus a `ResourceBag` value object
with arithmetic helpers used everywhere bank / circle / cost math happens.

## Files

- Create `src/game/resources/types.ts`.
- Create `src/game/resources/bag.ts`.
- Create `tests/resources/bag.test.ts`.

## API

```ts
// types.ts
export const RESOURCES = [
  'gold', 'wood', 'stone', 'steel', 'horse',
  'food', 'production', 'science', 'happiness',
  'worker',
] as const;
export type Resource = typeof RESOURCES[number];

export type ResourceBag = Record<Resource, number>;

export const EMPTY_BAG: Readonly<ResourceBag>;

// bag.ts — pure functions
export const bagOf = (entries: Partial<ResourceBag>): ResourceBag;
export const add = (a: ResourceBag, b: Partial<ResourceBag>): ResourceBag;
export const sub = (a: ResourceBag, b: Partial<ResourceBag>): ResourceBag; // throws on underflow
export const canAfford = (have: ResourceBag, cost: Partial<ResourceBag>): boolean;
export const total = (b: ResourceBag): number;
export const eq = (a: ResourceBag, b: ResourceBag): boolean;
```

## Steps

1. The resource list mirrors game-design.md §Tokens (`Gold, Wood, Stone,
   Steel, Worker, Horse, …`) plus the abstract goods used by buildings JSON
   (`food`, `production`, `science`, `happiness`).
2. `EMPTY_BAG` is `Object.freeze({ gold: 0, wood: 0, ... })`.
3. `add` / `sub` accept a partial second argument so call sites can write
   `sub(bag, { gold: 3 })`.
4. `sub` throws `RangeError` on underflow with the offending resource — never
   silently clamps.
5. Bags are immutable values; helpers always return a new object.

## Tests

- Identity: `add(EMPTY_BAG, EMPTY_BAG)` deep-equals `EMPTY_BAG`.
- `sub(bagOf({gold:3}), {gold:2})` → `bagOf({gold:1})`.
- `sub(bagOf({gold:1}), {gold:2})` throws `RangeError` mentioning `gold`.
- `canAfford(bagOf({gold:5,wood:1}), {gold:3,wood:1})` is `true`;
  with `wood:2` is `false`.
- `total` sums every resource.
