# 08.6 ‚Äî Tech card content (initial pass)

**Parent:** [08-cross-cutting.md](08-cross-cutting.md)

## Goal

Author the first batch of tech-card effects so the cards distributed by
[05.3](05.3-complete-move.md) actually do interesting things in the
recipient's hand. AI-assisted authoring; hand-curated.

## Files

- Modify [../src/data/technologies.json](../src/data/technologies.json) ‚Äî
  fill in the empty `cost`, `buildings`, `units`, `*Event` fields.
- Modify `src/data/schema.ts` ‚Äî refine `TechnologyDef` so the new fields
  are typed (cost as `Partial<ResourceBag>`, effects as
  `EventEffect[] | string` etc.).
- Create `src/game/tech/effects.ts` ‚Äî applies tech effects when a tech
  card is held / activated.
- Create `tests/tech/effects.test.ts`.

## API

```ts
export interface TechEffect {
  passive?: EventEffect[];   // applies as long as the holder has the card
  onAcquire?: EventEffect[]; // applies once when distributed
  onPlay?: EventEffect[];    // when explicitly played via <role>PlayTech
}

export const applyTechOnAcquire: (G: SettlementState, holder: PlayerID, tech: TechnologyDef) => void;
export const techPassives: (G: SettlementState, holder: PlayerID) => EventEffect[];

// One PlayTech move per role. Each gates by ownership of the role color
// of tech (red‚ÜíForeign, gold‚ÜíChief, green‚ÜíDomestic, blue‚ÜíScience).
export const chiefPlayTech:    Move<SettlementState>;
export const sciencePlayTech:  Move<SettlementState>;
export const domesticPlayTech: Move<SettlementState>;
export const foreignPlayTech:  Move<SettlementState>;
// args: cardID: string, payload?: unknown (for awaitInput effects)
```

## Authoring approach (AI-assisted)

1. Group [technologies.json](../src/data/technologies.json) entries by
   branch (Exploration, Fighting, Education, Civic, ‚Ä¶) and target color
   (red ‚Üí Foreign, gold ‚Üí Chief, green ‚Üí Domestic, blue ‚Üí Science).
2. Feed an LLM the branch + color + name and ask for one of three effect
   shapes:
   - **Passive resource bonus** (e.g. "Driving" ‚Üí +1 horse per turn).
   - **Unlock** (e.g. "Big guns" ‚Üí enables a higher-tier unit recruit).
   - **One-shot bonus** (e.g. "Loot car" ‚Üí gain 3 gold on acquire).
3. Hand-curate for power balance; rough cap = "+2 of any single
   resource per turn equivalent" per card.
4. Encode in the JSON, validating against the refined schema.

## Steps

1. Refine `TechnologyDef` schema to include typed effect fields.
2. Implement `applyTechOnAcquire` ‚Äî called from
   [05.3](05.3-complete-move.md) when a tech card is distributed.
3. Implement `techPassives` ‚Äî composed into the existing modifier
   pipeline (08.2's `hasModifierActive` reads from this list too).
4. **Per-role PlayTech moves.** Each lives next to its role's other
   moves: `src/game/roles/<role>/playTech.ts`. Each:
   - Validates caller seat holds the matching role.
   - Validates `cardID` is in the seat's tech hand for that color.
   - Validates `tech.onPlay` is non-empty (otherwise nothing to play).
   - Dispatches via 08.2's dispatcher with `tech.onPlay`.
   - Marks the card as "played" ‚Äî V1 model: tech cards persist after
     play (passive stays); only `onPlay` effects are one-shot.
5. The four event-card colors and the tech-card effects share the
   `EventEffect` taxonomy from 08.2 ‚Äî no new effect kinds unless content
   genuinely demands it.

## Tests

- Acquiring a tech with `onAcquire: [{ kind: 'gainResource', ... }]`
  credits the target.
- A passive effect from a held tech is observable via `techPassives`.
- Schema rejects a tech card with an unknown effect kind.
- All entries in `technologies.json` parse without throwing.
- `chiefPlayTech(goldTechWithOnPlay)` fires the `onPlay` effects exactly
  once; the passive remains active afterward.
- Wrong-role play: `domesticPlayTech` on a red tech ‚Üí `INVALID_MOVE`.
- A tech with empty `onPlay` rejects `*PlayTech` with `INVALID_MOVE`.

## Dependencies

01.2, 05.3, 08.2.

## Out of scope

- Removing / trading tech cards. They're keepsakes for V1.
- Tech-card cross-references that require a second card to activate.
- "Once-per-game" `onPlay` cards beyond a simple "played" flag. If we
  want true single-use techs later, add a `consumedOnPlay: true` field
  and have the move drop the card.
