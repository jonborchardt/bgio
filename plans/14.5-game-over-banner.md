# 14.5 — Game-over banner

**Parent:** [14-playtest-followups.md](14-playtest-followups.md)

## Goal

When `endIf` (08.5) fires, bgio stores the `GameOutcome` on
`ctx.gameover`. Nothing in `Board.tsx` reads that today, so the UI
goes silent at game end — see playtest issue V6.

## Files

- Create `src/ui/layout/GameOverBanner.tsx`.
- Modify `src/Board.tsx` — render `<GameOverBanner>` when
  `ctx.gameover` is truthy.

## API

```tsx
import type { GameOutcome } from '../../game/endConditions';

export interface GameOverBannerProps {
  outcome: GameOutcome; // { kind: 'win' | 'timeUp', turns, settlementsJoined }
}
export const GameOverBanner: React.FC<GameOverBannerProps>;
```

## Steps

1. Banner renders a sticky strip at the top of the board (above
   `BoardShell`) with one of:
   - `'win'`: "You won! Settlements joined: 10 — turns: N"
   - `'timeUp'`: "Time's up at turn N. Settlements joined: M / 10"
2. Use `palette.role.chief` accent for the win banner; muted for
   timeUp.
3. While the banner is up, all role panels and the chief end-turn
   buttons stay rendered but don't dispatch — bgio ignores moves
   after `gameover` anyway, so the gating is just visual.
4. Add a "Play again" button (per game-design.md §End — V1 just
   reloads the page; later 10.x lobby `playAgain` plumbing
   replaces this).

## Tests

- Pure render: a stub `outcome={{kind:'win',turns:42,settlementsJoined:10}}`
  renders the win text.
- `<Board>` smoke: with a fixture state where `ctx.gameover` is
  truthy, the banner is in the DOM and "End my turn" actions are
  hidden / disabled.

## Dependencies

08.5 (endIf + GameOutcome), 09.1 (Board layout).

## Out of scope

- Personal best lookup ("you beat your best by 4 turns") — that
  needs the server-side runs table from 10.7 and the
  `/users/me/runs` endpoint, which we haven't mounted yet.
