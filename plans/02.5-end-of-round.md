# 02.5 — End-of-round hook registry

**Parent:** [02-engine.md](02-engine.md)

## Goal

Let any later module register a function that runs at `endOfRound.onBegin`,
without those modules importing each other. Avoids a god-function and keeps
modules independent.

## Files

- Create `src/game/hooks.ts`.
- Modify `src/game/phases/endOfRound.ts` to consume the registry.
- Create `tests/hooks.test.ts`.

## API

```ts
import type { SettlementState } from './types';

export type RoundEndHook = (
  G: SettlementState,
  ctx: Ctx,
  random: RandomAPI,
) => void; // Immer-mutating, like all bgio moves

// Hooks are registered at module evaluation time (top-level of each role
// file) into a frozen list assembled before Settlement is exported.
export const registerRoundEndHook: (name: string, hook: RoundEndHook) => void;

// Called by endOfRound.onBegin.
export const runRoundEndHooks: (
  G: SettlementState, ctx: Ctx, random: RandomAPI,
) => void;

// Test helper.
export const __resetHooksForTest: () => void;
```

## Steps

1. Module-internal `Map<string, RoundEndHook>` so the same name can re-register
   (idempotent — same name overwrites; cross-module name collisions throw).
2. `runRoundEndHooks` iterates in insertion order. Each hook is wrapped in a
   try/catch that surfaces `{ hook: name, error }` so a buggy hook doesn't
   silently corrupt state.
3. Provide an `__resetHooksForTest` for tests to start from a clean slate.
4. The first real consumer is 03.3 (mat-sweep at end of round). Until then,
   `runRoundEndHooks` is a no-op.

## Tests

- Registering two hooks runs both, in registration order.
- Registering with a duplicate name throws; with the same name to overwrite
  works (call site clears via `__resetHooksForTest` first).
- A throwing hook surfaces the error and does not block subsequent hooks.
