# 10.1 — Server package layout

**Parent:** [10-multiplayer.md](10-multiplayer.md)

## Goal

Carve out a server entry point that hosts the same `Settlement` Game
definition the client imports, without splitting the repo into a monorepo
unless we have to.

## Files

- Create `server/index.ts` — bgio Koa server bootstrap.
- Create `server/tsconfig.json` — extends root, but `module: nodenext` and
  excludes the React app.
- Modify root `package.json`: add `server` scripts (`server:dev`,
  `server:build`, `server:start`).
- Modify [tsconfig.app.json](../tsconfig.app.json) so the server tree is
  excluded from the client build.
- Create `server/Dockerfile` — placeholder; full deploy in 13.2.
- Create `tests/server/boot.test.ts`.

## API

```ts
// server/index.ts
import { Server, FlatFile } from 'boardgame.io/server';
import { Settlement } from '../src/game';

export const createServer = (opts?: { port?: number; storage?: any });
```

## Steps

1. Resolve the cross-import question: Vite's `src/` is consumed by Node
   for the server. Two options:
   - (chosen) bundle the server with `tsc` and rely on shared TS via
     `composite` references, since `verbatimModuleSyntax` already covers
     us.
   - alternative: pnpm workspace; deferred to a sub-plan revision if the
     simple shared-source approach hurts.
2. `createServer` constructs a bgio `Server` with the `Settlement` game and
   storage from 10.4 (default `FlatFile`).
3. Add `npm run server:dev` running `tsx server/index.ts` (dev dep
   `tsx` added).
4. The client side change is zero — `App.tsx` doesn't know whether it's
   networked yet. 10.2 wires the client transport.

## Tests

- `createServer({ port: 0 })` boots and reports a listening port.
- Hitting `/games/settlement/create` (bgio's lobby endpoint) returns a
  match ID.
