# 06.4 — domesticProduce move

**Parent:** [06-domestic.md](06-domestic.md)

## Goal

Sum the parsed yields of every in-play building (plus worker bonuses) and
deposit the resources into the bank, per game-design.md §Domestic
"Calculate all produced goods …".

## Files

- Create `src/game/roles/domestic/produce.ts`.
- Register in `src/game/moves.ts`.
- Create `tests/roles/domestic/produce.test.ts`.

## API

```ts
export const domesticProduce: Move<SettlementState>;
// no args; idempotent within a round (guarded by per-round flag)
```

## Steps

1. Validate caller seat holds `domestic` and is in `domesticTurn`.
2. Validate `G.domestic.producedThisRound !== true`. (Idempotency.)
3. For each grid cell:
   - Parse `BuildingDef.benefit` via 06.3 (cache parses across the move).
   - Add `parsed.resources` to a running yield.
   - If a worker token is on the cell: add the worker bonus
     (game-design.md "if a building has a worker, the worker goods are in
     addition to the normal default goods" — V1 worker bonus = same as
     default; can be overridden by future `WorkerBonusDef` data).
   - `parsed.effects` are not applied here; they affect *other* moves
     (Foreign upkeep, recruit cost, combat).
4. `transfer(EMPTY_BAG, G.bank, totalYield)` — there's no source bag,
   the bank just gains. Implementation: directly `add` into bank.
5. Set `G.domestic.producedThisRound = true`.
6. Round-end hook (registered here): clears `producedThisRound`.

## Tests

- Empty grid yields zero resources to bank.
- 1 Granary in grid (`"2 food"`) yields 2 food into bank.
- Mill (`"2 food and 1 production"`) yields both.
- Producing twice in a round: second call returns `INVALID_MOVE`.
- After `endOfRound`, the flag resets and produce works again.
- Worker on a building doubles its yield (V1 stub).

## Dependencies

01.2, 03.2, 06.1, 06.3, 02.5.

## Out of scope

- Adjacency bonuses (06.5).
- Per-resource caps (none in V1).
