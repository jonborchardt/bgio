# 01.5 — Refresh CLAUDE.md

**Parent:** [01-foundations.md](01-foundations.md)

## Goal

Update [../CLAUDE.md](../CLAUDE.md) so future sessions read the new module
layout, conventions, and project stance. CLAUDE.md is loaded into every
session's context — drift here is expensive.

## Files

- Modify [../CLAUDE.md](../CLAUDE.md).

## Steps

1. Replace the "Layout you should know" section with the new tree:
   - `src/game/` — barrel; exports `Settlement`, types, moves.
   - `src/game/types.ts`, `src/game/random.ts`, `src/game/playerView.ts`,
     `src/game/hooks.ts`, `src/game/endConditions.ts`.
   - `src/game/phases/{chief,others,endOfRound,stages}.ts`.
   - `src/game/resources/`, `src/game/events/`, `src/game/opponent/`,
     `src/game/ai/`, `src/game/plugins/`.
   - `src/game/roles/{chief,science,domestic,foreign}/` — each owns its
     moves and any role-local helpers.
   - `src/data/` — JSON content + typed loaders.
   - `src/ui/{layout,cards,resources,mat,deck,hand,chief,science,domestic,foreign,chat}/` —
     React components, MUI primitives only.
   - `server/` — bgio Koa server + storage adapters.
   - `tests/` — mirrors `src/` shape; `tests/helpers/` for shared factories.
2. Replace the "When changing the game" recipe with the role-module
   workflow: pick the role's folder, change types + moves + tests, then
   the matching panel under `src/ui/<role>/`.
3. Add a "Project stance" section listing:
   - Network play is the primary delivery; hot-seat is a fallback.
   - Default game is 4-player with 3 bots.
   - **No fail mode** — only a win condition.
   - Codename "Settlement"; rename pass deferred until the user picks
     a real name.
4. Keep the "UI + styling rules" section but extend it: theme tokens
   from 09.4 must be used (no raw hex literals); per-role accents come
   from `palette.role.<role>`.
5. Update the "Common commands" list to include
   `npm run server:dev`, `build:hotseat`, `build:networked`,
   `e2e:smoke`, `test:coverage`.
