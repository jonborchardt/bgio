# 14.13 — Role panel "done" state polish

**Parent:** [14-playtest-followups.md](14-playtest-followups.md)

## Goal

Once a non-chief seat clicks its panel's "End my turn" button (the
14.2 `<role>SeatDone` move), the button should reflect that the
seat has flipped done — disable, swap copy to "Turn ended". Today
the button stays enabled because the per-panel `canAct` only checks
the active stage; calling the move again is a no-op but the
unchanged UI gives no feedback.

## Why

Found by the post-14.1 / 14.2 hot-seat smoke. Re-clicking the
button after ending the turn is harmless (the move re-flips an
already-true flag) but the user has no feedback that "yes, you
ended your turn — switch to the next seat / wait for them".

## Files

- Modify `src/ui/science/SciencePanel.tsx` — gate the End-my-turn
  button on `!alreadyDone`.
- Modify `src/ui/domestic/DomesticPanel.tsx` — same.
- Modify `src/ui/foreign/ForeignPanel.tsx` — same.

## Steps

1. Each panel reads `alreadyDone = G.othersDone?.[playerID] === true`.
2. End-my-turn button: `disabled = !canAct || alreadyDone`,
   label flips to "Turn ended" while disabled. Mirrors the
   already-shipped pattern on DomesticPanel's Produce button
   ("Produced" when `alreadyProduced`).

## Tests

Existing `it.todo` slots in `tests/ui/<role>/*.test.tsx` cover the
"End my turn" button — the new behavior is folded into them when
RTL eventually lands. No new tests today.

## Dependencies

14.2 (the moves + buttons), 09.1 (panel layout), 09.4 (theme tokens
the buttons use).

## Out of scope

- Auto-advancing the seat picker to the next pending seat after
  done. Deliberate: the user may want to revisit a seat for
  bookkeeping (e.g. inspect the science grid the chief just
  contributed to).
- A "clear my done flag" move. V1 keeps done one-way; the round-end
  hook resets it.
