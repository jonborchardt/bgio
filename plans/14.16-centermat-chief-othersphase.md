# 14.16 — CenterMat hidden in othersPhase from chief seat (multi-role)

**Parent:** [14-playtest-followups.md](14-playtest-followups.md)

## Goal

Stop hiding the CenterMat in cases where the ChiefPanel is NOT
actually rendering. Today the gate uses
`!expanded('chief')` (14.3); `expanded` only checks "local seat
holds chief", not "ChiefPanel will actually render". In `othersPhase`,
ChiefPanel returns null (its own gating). So the chief seat sees
neither the chief panel nor the center mat — losing the only
view of every seat's circle contents.

## Why

Found by post-14.3 review of [`src/Board.tsx`](../src/Board.tsx).
Worst case: 2-player game where seat 0 holds `chief + science`.
In `othersPhase` `scienceTurn`, seat 0 needs to see the center
mat to know what gold the chief just distributed; today it's
hidden.

## Files

- [`src/Board.tsx`](../src/Board.tsx)

## Steps

1. Replace `const showCenterMat = !expanded('chief');` with the
   actual ChiefPanel-visible condition:
   `const chiefPanelVisible = expanded('chief') && ctx.phase === 'chiefPhase';`
   `const showCenterMat = !chiefPanelVisible;`
2. Verify in dev that:
   - 4-player chiefPhase from chief seat → no CenterMat (correct;
     ChiefPanel shows circles).
   - 4-player othersPhase from chief seat → CenterMat visible
     (the chief is just spectating their own and others' circles
     during othersPhase).
   - 2-player othersPhase from seat 0 (chief+science) → CenterMat
     visible.

## Out of scope

- Showing two views of the same circles when both are visible —
  ChiefPanel's CircleEditors and CenterMat's Circles. The
  CenterMat is a smaller summary; the duplication is
  intentional in non-chief-phase contexts.
