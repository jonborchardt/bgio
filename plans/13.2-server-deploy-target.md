# 13.2 — Server deploy target

**Parent:** [13-deploy.md](13-deploy.md)

## Goal

Pick a host for the bgio server, write a Dockerfile, and add a deploy
workflow. Keep cost and ops minimal — this is a hobby project.

## Files

- Modify `server/Dockerfile` (placeholder created in 10.1).
- Create `.github/workflows/deploy-server.yml`.
- Create `server/README.md` documenting the host choice.

## Host: Render

Locked in. Reasons:
- Persistent disk for SQLite (10.4 / 13.3).
- Push-from-git deploy is the simplest possible CD.
- Long-lived Node process — bgio's `Server` expects this; rules out
  Cloudflare Workers / similar request-scoped runtimes.
- Free tier suits a hobby project; easy upgrade path if traffic grows.

**Free-tier sleep accepted.** Render's free web services sleep after
~15 min idle and take ~30s to wake. The 10.6 server-down spinner with
backoff covers the wake window — first visitor of the day eats the cold
start, subsequent traffic is warm. We don't pay $7/mo to avoid it. If
this proves painful in real use, upgrade is a config change, not a
rewrite.

Alternatives (Fly.io, Railway, self-host) remain viable if Render becomes
unattractive — the only Render-specific bits in our code are the
`render.yaml` and the deploy workflow, both small.

## Steps

1. `Dockerfile`: multi-stage Node 20 → run `npm ci && npm run server:build
   && npm run server:start`.
2. Health endpoint: bgio's `Server` exposes `/`; we add a `/healthz` for
   the host.
3. `deploy-server.yml`:
   - Trigger: `push` to `main` AND files under `server/` or `src/game/**`.
   - Render's preferred path is git-driven via `render.yaml` (no separate
     image push needed). Add `render.yaml` to the repo root or `server/`
     and let Render auto-deploy. The workflow's job is to fail loudly on
     a server build break before Render attempts deploy.
4. Document setup steps in `server/README.md` (env vars, secrets).

## Tests

- Local `docker build` succeeds.
- A smoke check in CI hits `/healthz` after starting the container.

## Dependencies

10.1, 10.4 (storage chosen), 12.4 (CI).

## Out of scope

- Multi-region. Single region for V1.
- Autoscaling.
