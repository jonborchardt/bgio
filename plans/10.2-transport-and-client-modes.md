# 10.2 — Client transport modes

**Parent:** [10-multiplayer.md](10-multiplayer.md)

## Goal

`App.tsx` selects between hot-seat (local) and networked transport at build
time, via env var. Hot-seat continues to be the GH Pages default; networked
is opt-in.

## Files

- Modify `src/App.tsx`.
- Create `src/clientMode.ts` — env detection + transport factory.
- Modify [vite.config.ts](../vite.config.ts) — define `import.meta.env`
  defaults.
- Modify `.env.example` (create if missing) — document the vars.
- Create `tests/clientMode.test.ts`.

## API

```ts
// src/clientMode.ts
export type ClientMode = 'hotseat' | 'networked';

export const detectMode = (): ClientMode;
// reads import.meta.env.VITE_CLIENT_MODE; defaults to 'hotseat'.

export const networkedClientFactory: (matchID: string, playerID: string,
  credentials: string) => ClientType;
```

## Steps

1. `App.tsx` calls `detectMode()`. If `'hotseat'`, uses bgio's `Local`
   transport (current behavior). If `'networked'`, expects the lobby
   (10.3) to provide `matchID`, `playerID`, `credentials` via context.
2. `vite.config.ts` defines `VITE_CLIENT_MODE` and
   `VITE_SERVER_URL` defaults.
3. `.env.example`:
   ```
   VITE_CLIENT_MODE=hotseat
   VITE_SERVER_URL=http://localhost:8000
   ```
4. The hot-seat smoke test from earlier stages still passes — no behavior
   change in that mode.

## Tests

- `detectMode()` returns `'hotseat'` when `VITE_CLIENT_MODE` unset.
- With `VITE_CLIENT_MODE=networked`, `App.tsx` constructs the SocketIO
  transport.
- Hot-seat smoke test still passes.
