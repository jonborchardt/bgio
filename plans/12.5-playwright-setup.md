# 12.5 — Playwright setup

**Parent:** [12-testing-debug.md](12-testing-debug.md)

## Goal

`@playwright/test` configured to drive the running dev server. One smoke
spec that boots the app and verifies the board renders. Foundation for
future multi-browser hot-seat / multiplayer flows.

## Files

- Create `playwright.config.ts`.
- Create `tests-e2e/smoke.spec.ts`.
- Modify `package.json`: `e2e:smoke`, `e2e:install`.
- Add `@playwright/test` as devDependency.

## Steps

1. `playwright.config.ts`:
   ```ts
   export default defineConfig({
     webServer: { command: 'npm run dev', port: 5173, reuseExistingServer: true },
     use: { baseURL: 'http://localhost:5173' },
     projects: [{ name: 'chromium', use: devices['Desktop Chrome'] }],
   });
   ```
2. `smoke.spec.ts`:
   ```ts
   test('board renders', async ({ page }) => {
     await page.goto('/');
     await expect(page.getByTestId('board-shell')).toBeVisible();
   });
   ```
3. Add `data-testid="board-shell"` to `<BoardShell>` from 09.1.
4. CI step `npm run e2e:smoke` (12.4 already lists this).
5. Document `npm run e2e:install` for the local-dev "install browsers"
   step.

## Tests

- The smoke spec passes locally and in CI.

## Dependencies

09.1 (BoardShell), 12.4 (CI step).

## Out of scope

- Multi-browser tests for networked play. Add as a sub-plan once 10.x is
  fully landed.
- Visual regression. A separate sub-plan can add screenshot baselines.
