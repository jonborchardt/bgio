# 14.11 — End-to-end networked playtest

**Parent:** [14-playtest-followups.md](14-playtest-followups.md)

## Goal

Validate the networked path with two browser tabs against
`npm run dev:full`. The 2026-04-29 hot-seat playtest didn't exercise
networked mode at all — see [../notes/playtest-2026-04-29.md](../notes/playtest-2026-04-29.md)
issue O2. This sub-plan is a playtest, not code: produce a
follow-up `notes/playtest-*.md` with findings.

## Files

- (no source code — produces a notes file)
- Create `notes/playtest-networked-<date>.md` after running.

## Steps

1. Pre-flight: `npm install` on the test host so `concurrently`,
   `better-sqlite3`, `@playwright/test`, and `@vitest/coverage-v8`
   are available. (See the 13.x deploy notes — these are declared
   as deps but the install step has been deferred.)
2. Run `npm run dev:full` (server on :8000 + client on :5179).
3. Tab A: open `http://localhost:5179/?VITE_CLIENT_MODE=networked`
   (or build with `VITE_CLIENT_MODE=networked` and serve).
4. Register `alice` / `password` via AuthForms. Confirm the lobby
   list loads.
5. Tab B: register `bob` / `password`. Confirm separate session.
6. Tab A creates a 2-player match, joins seat 0.
7. Tab B joins seat 1 from the lobby list.
8. Drive a round: chief distributes, both seats run their stage,
   end-of-round fires.
9. Capture findings: missing UI, broken transitions, dispatcher
   gaps, server logs, console errors, REST 4xx/5xx.
10. Write `notes/playtest-networked-<date>.md` listing
    Blocker / Important / Nit issues with screenshots and a
    plan-input punch list (the same shape as the hot-seat notes).

## Tests

- The notes file exists and is non-empty.
- Each blocker has a clear `where:` (file path + line) and a
  proposed fix shape.

## Dependencies

10.x (server, lobby, accounts), 11.7 (solo if exercised), 12.7
(`dev:full`), 13.3 (SQLite when `npm install` lands), 14.1 + 14.2
(without these, even networked mode probably can't progress past
chief turn).

## Out of scope

- Any actual code fix. This sub-plan is read-only: it produces a
  notes file. New sub-plans land afterward to address findings.
- Multi-region or load testing.
