# 04.4 — Gold event (stub)

**Parent:** [04-chief.md](04-chief.md)

## Goal

Declare the move surface for the chief playing one gold-color event card,
gated behind 08 (event content & dispatcher).

## Files

- Create `src/game/roles/chief/playGoldEvent.ts`.
- Register move in `src/game/moves.ts`.
- Create `tests/roles/chief/playGoldEvent.test.ts`.

## API

```ts
export const chiefPlayGoldEvent: Move<SettlementState>;
// args: cardID: string
```

## Steps

1. Validate caller holds chief.
2. Validate `cardID` is in the chief's event hand
   (`G.events.hands.gold[ctx.playerID]`).
3. Validate the chief has not already played a gold event this round
   (per game-design.md "may play 1 ... at any time").
4. Look up the card via 08.1's loader; dispatch via 08.2's dispatcher.
5. Mark the card "used" in the per-player event-cycle ledger (08.1).
6. Set `G._eventPlayedThisRound.chief = true`.
7. Stage interrupt: call `enterEventStage(events, prev)` if effects need
   user follow-up; otherwise resolve atomically.

**Until 08 lands**, the move body short-circuits: `if (!G.events) return
INVALID_MOVE;`. Implementing the card surface is an 08.3 follow-up.

## Tests

- Without 08 landed: move returns `INVALID_MOVE`.
- With a stub event hand + dispatcher mock: card is consumed, dispatcher
  called once, flag set.
- Playing a second gold event in the same round returns `INVALID_MOVE`.
- Wrong color (e.g. blue card in chief's hand) returns `INVALID_MOVE`.

## Dependencies

01.1, 02.1, 02.2 (event stage). Hard dep on 08 for real content.

## Out of scope

- The event card library itself (08.1).
- Effects (08.2 dispatcher).
