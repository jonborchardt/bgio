# 10.4 — Storage adapter

**Parent:** [10-multiplayer.md](10-multiplayer.md)

## Goal

Pick a storage adapter for `Server({ db })` per environment: bgio's
`FlatFile` in dev, a SQLite adapter in production. We do not lock in
Postgres until 13.3.

**bgio-first stance:** every adapter implements bgio's existing `Async`
storage interface (`boardgame.io/internal` / `boardgame.io/storage`).
Do **not** introduce a parallel `Storage` type — bgio's interface is the
contract, and any community adapter (`bgio-postgres`, `bgio-sqlite`,
custom) plugs into the same `db` slot. We only write a small factory.

## Files

- Create `server/storage/index.ts` — factory only; no new interface.
- Create `server/storage/sqlite.ts` — implements bgio's `Async` against
  `better-sqlite3` (only if no maintained community adapter fits). Use
  `bgio-sqlite` if it works for our schema; otherwise this file
  implements the four `Async` methods (`createMatch`, `setState`,
  `fetch`, `wipe`, `listMatches`, `setMetadata`) directly.
- Modify `server/index.ts` to construct storage via the factory.
- Create `tests/server/storage.test.ts`.

## API

```ts
import type { Async } from 'boardgame.io/internal';
import { FlatFile } from 'boardgame.io/server';

// Factory returns bgio's Async directly — no wrapper interface.
export const makeStorage: (
  kind: 'flatfile' | 'sqlite',
  opts: { dir?: string; dbPath?: string },
) => Async;
```

## Steps

1. `flatfile`: `new FlatFile({ dir: opts.dir ?? './bgio-data' })` —
   bgio ships this; we don't wrap it.
2. `sqlite.ts`: prefer the maintained `bgio-sqlite` adapter
   (`new SqliteStore({ database: opts.dbPath })`). Only hand-roll an
   `Async` implementation if no adapter matches our schema needs.
   When hand-rolled, the schema is whatever bgio expects of `Async`:
   ```
   matches(id PK, gameName, setupData, players JSON, gameState JSON, ...)
   metadata(id PK, ...)
   ```
3. `makeStorage` is a `switch (kind)` returning the appropriate `Async`.
4. Server passes it straight in: `new Server({ games, db: makeStorage(...) })`.

## Tests

- `makeStorage('flatfile', ...)` returns an object satisfying bgio's
  `Async` (smoke check on method shape).
- Round-trip a match through `flatfile`: `createMatch` → `setState` →
  `fetch`.
- `sqlite` round-trip with an in-memory db (`:memory:`).
- Switching storage between two server boots preserves matches in flatfile.
