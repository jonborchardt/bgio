# 05.5 — Science UI panel

**Parent:** [05-science.md](05-science.md)

## Goal

Render the 3×3 science grid with per-card cost/paid bars, a contribute UI,
and a "Complete" button enabled only when paid ≥ cost.

## Files

- Create `src/ui/science/SciencePanel.tsx`.
- Create `src/ui/science/ScienceCard.tsx` — single card view (cost bar,
  paid bar, contribute control, complete button).
- Create `src/ui/science/UnderCardsPopover.tsx` — peek at the 4 tech cards
  underneath.
- Modify `src/Board.tsx` to render `<SciencePanel>` when local seat holds
  science.
- Create `tests/ui/science/SciencePanel.test.tsx`.

## API

```tsx
export const SciencePanel: React.FC<BoardProps<SettlementState>>;
```

## Steps

1. Read `props.G.science.grid`. Render 3 columns; row 0 closest to player.
2. Each `<ScienceCard>` shows:
   - color & level
   - per-resource cost & paid amount (use `<ResourceBag>` from 09.3 once
     ready; until then, simple chips)
   - "Contribute" affordance: pop a resource picker bound to the seat's
     wallet
   - "Complete" button: enabled when `canAfford(paid, cost)`
   - "Peek" button → `<UnderCardsPopover>`
3. Lowest-first rule visualized: cards above the lowest unfinished card in
   the column are dimmed and reject clicks.
4. Theme: per-color column accents from `palette.science.{red,gold,green,blue}`
   (added in 09.4).

## Tests

- 3×3 grid renders for a fresh game.
- Clicking "+gold" on a paid bar fires `scienceContribute(id, {gold:1})`.
- "Complete" disabled until paid ≥ cost.
- Higher-level card in the same color is dimmed when a lower-level card
  is unfinished.

## Dependencies

05.1, 05.2, 05.3, 09.2 (card components — soft dep until then).

## Out of scope

- Animations on completion.
- Drag-to-fill resource pours; explicit "+1" stepper is enough for V1.
