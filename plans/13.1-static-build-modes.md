# 13.1 — Static build modes

**Parent:** [13-deploy.md](13-deploy.md)

## Goal

A single SPA codebase produces two `dist/` flavors based on env:
hot-seat (default) and networked. Build-time env toggle, no runtime
fork.

## Files

- Modify [vite.config.ts](../vite.config.ts).
- Modify `package.json`:
  - `build:hotseat` (default — what existing GH Pages workflow uses).
  - `build:networked` (sets `VITE_CLIENT_MODE=networked`).
- Modify [.github/workflows/deploy-pages.yml](../.github/workflows/deploy-pages.yml)
  to run `build:hotseat`.

## Steps

1. `vite.config.ts` reads `process.env.VITE_CLIENT_MODE` (already in 10.2).
   No additional config required there; just ensure both modes are
   exercised in CI.
2. `build:networked` script:
   ```
   VITE_CLIENT_MODE=networked VITE_SERVER_URL=$SERVER_URL vite build
   ```
3. `build:hotseat` is the existing `build` script.
4. The deploy-pages workflow is unchanged in behavior — it builds the
   hot-seat flavor.

## Tests

- `npm run build:hotseat` produces `dist/` that boots in `Local`
  transport (verify via Playwright smoke spec — already in 12.5).
- `npm run build:networked` produces `dist/` that, when served, routes to
  the lobby first.
