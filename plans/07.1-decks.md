# 07.1 — Battle & Trade decks

**Parent:** [07-foreign.md](07-foreign.md)

## Goal

Build the Battle and Trade decks per game-design.md §Setup.Foreign:
"Sort by number, randomize each pile of the same number, stack with highest
numbers at the bottom." Plus: redact deck order in `playerView`.

## Files

- Create `src/data/battleCards.json`, `src/data/tradeCards.json` (empty arrays
  to start; content lands separately).
- Create `src/data/decks.ts` — typed loader + schema.
- Create `src/game/roles/foreign/decks.ts` — setup helper.
- Create `tests/roles/foreign/decks.test.ts`.

## API

```ts
export interface BattleCardDef {
  id: string;
  number: number;          // sort key
  units: BattleUnit[];     // enemy units on this card
  reward?: ResourceBag;    // success reward
  failure?: { tribute: ResourceBag };  // failure cost
  flavor?: string;
}

export interface TradeCardDef {
  id: string;
  number: number;
  required: ResourceBag;
  reward: ResourceBag;
  flavor?: string;
}

export const buildBattleDeck: (random: RandomAPI) => BattleCardDef[];
export const buildTradeDeck: (random: RandomAPI) => TradeCardDef[];
```

## Steps

1. Schema validators mirror 01.2.
2. `buildBattleDeck`:
   - Group cards by `number`.
   - Shuffle within each group via `random.Shuffle`.
   - Concatenate groups in ascending order (top of deck = lowest number).
   - "Stack with highest numbers at the bottom" matches: low numbers
     drawn first.
3. `buildTradeDeck` — same algorithm.
4. Setup stores them at `G.foreign.battleDeck` and `G.foreign.tradeDeck`.
5. **Redaction:** 02.4's `playerView` uses `redactDeckOrder` for both decks
   when the viewer doesn't hold Foreign.

## Tests

- Empty content arrays produce empty decks (sanity).
- A fixture deck of `[A1, B1, C2]` (numbers shown) is sorted with both
  number-1 cards before number-2.
- Redaction: `playerView` for a non-Foreign seat sees `null[]` for both
  decks, of correct length.
- Same seed → same deck order.
