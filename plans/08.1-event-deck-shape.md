# 08.1 — Event deck shape & per-player cycle state

**Parent:** [08-cross-cutting.md](08-cross-cutting.md)

## Goal

Define the data shape for event cards, load four per-color decks of 4
starter cards each (per game-design.md §Events: "Each get 4 events"), and
track per-player cycle state so the "must choose one each round, can't
repeat until cycle resets" rule works.

## Files

- Create `src/data/events.json`.
- Create `src/data/events.ts` — typed loader.
- Create `src/game/events/state.ts` — cycle bookkeeping.
- Modify `src/game/setup.ts` to call `setupEvents`.
- Create `tests/events/cycle.test.ts`.

## API

```ts
export type EventColor = 'gold' | 'blue' | 'green' | 'red';

export interface EventCardDef {
  id: string;
  color: EventColor;
  name: string;
  effects: EventEffect[];   // typed, see 08.2
}

export interface EventsState {
  decks: Record<EventColor, EventCardDef[]>;     // never depleted; cycle when used
  hands: Record<EventColor, Record<PlayerID, EventCardDef[]>>; // visible options
  used: Record<EventColor, Record<PlayerID, string[]>>; // used IDs in current cycle
  playedThisRound: Record<PlayerID, EventColor[]>;
}

export const setupEvents: (
  assignments: Record<PlayerID, Role[]>,
  random: RandomAPI,
) => EventsState;

export const cycleAdvance: (state: EventsState, color: EventColor, seat: PlayerID) => void;
```

## Steps

1. `events.json` schema validates each card has `id`, `color`,
   `effects` (array of typed entries — schema for `effects` lives in 08.2;
   here we accept any array and let the dispatcher validate).
2. `setupEvents` deals 4 cards per color into the holding seat's `hands`
   bucket.
3. `cycleAdvance(color, seat)`:
   - Push the just-played card id into `used[color][seat]`.
   - If `used.length === 4` (the cycle is complete), clear it and the
     pool of 4 reopens.
4. The `playedThisRound` map gates "may play 1 ... at any time" per round;
   reset by an end-of-round hook registered here.

## Tests

- Setup deals 4 cards of each color into the right seat's hand.
- After playing 4 different cards in a color, the 5th allowed pick is
  any of the original 4 again (cycle reset).
- `playedThisRound` resets at end of round.
- Two seats play different cards; cycles tracked independently.
