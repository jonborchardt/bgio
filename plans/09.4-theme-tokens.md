# 09.4 — Theme tokens

**Parent:** [09-ui.md](09-ui.md)

## Goal

Extend [src/theme.ts](../src/theme.ts) with the design tokens the UI needs:
per-resource color, per-role accent, per-tier surface, per-event-color hue.
All consumed via `palette.<group>.<key>`. No raw hex literals in components
or panels.

## Files

- Modify `src/theme.ts`.
- Create `tests/theme.test.ts`.

## API addition

```ts
// Module augmentation
declare module '@mui/material/styles' {
  interface Palette {
    resource: Record<Resource, PaletteColor>;
    role: Record<Role, PaletteColor>;
    tier: Record<'beginner'|'intermediate'|'advanced', PaletteColor>;
    eventColor: Record<EventColor, PaletteColor>;
  }
  interface PaletteOptions extends Partial<Palette> {}
}
```

## Steps

1. Pick a ramp slot for each resource (gold = yellow, wood = brown, stone =
   grey, steel = slate, horse = orange, food = green, production = ochre,
   science = blue, happiness = pink, worker = teal). Create the ramp first
   per the existing `ramps` style in `theme.ts`.
2. Map `role`: chief = gold ramp, science = blue, domestic = green,
   foreign = red.
3. Map `tier`: beginner = light surface, intermediate = mid, advanced =
   dark border.
4. Map `eventColor`: gold/blue/green/red — same ramps as role colors but
   distinct semantic tokens (so we can change one without disturbing the
   other).
5. Update the existing `colors` semantic block to alias the new groups.

## Tests

- For each documented palette key, the resolved color is a valid hex
  string of length 7 (or 9 for alpha).
- `palette.resource.gold` !== `palette.resource.wood`.
- Module augmentation compiles (catch via `npm run typecheck`).

## Dependencies

03.1 (Resource enum), 01.1 (Role), 08.1 (EventColor). All these need to
exist as values for the augmentation to compile.

## Out of scope

- Dark / light mode toggle. V1 single mode.
