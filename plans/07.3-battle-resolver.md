# 07.3 ‚Äî Battle resolver (pure)

**Parent:** [07-foreign.md](07-foreign.md)

## Goal

A pure deterministic function that takes (player committed units, enemy
units on a battle card, ability tags) and returns a turn-by-turn battle log
plus the final outcome. Heavily unit-tested. Used by 07.4 inside a move.

## Files

- Create `src/game/roles/foreign/battleResolver.ts`.
- Create `src/game/roles/foreign/abilities.ts` ‚Äî typed ability tags.
- Create `tests/roles/foreign/battleResolver.test.ts` ‚Äî extensive table-driven.

## API

```ts
export interface ResolverInput {
  player: UnitInstance[];
  enemy: UnitInstance[];
  enemyDamageRule: 'attacksWeakest' | 'attacksStrongest' | 'attacksClosest' | 'attacksMostDamage';
  // damageAllocations: how the player wants to absorb each round's incoming
  // damage. One entry per round-of-incoming, in order.
  damageAllocations: ReadonlyArray<DamageAllocation>;
}

export interface DamageAllocation {
  // Map of player unit defID ‚Üí damage absorbed by that unit this round.
  // Sum must equal the incoming damage; per-unit absorption must be lethal
  // unless leftover < unit health.
  byUnit: Record<string, number>;
}

export interface ResolverOutput {
  log: BattleLogEvent[];
  outcome: 'win' | 'lose' | 'mid'; // 'mid' if input incomplete
  finalPlayer: UnitInstance[];
  finalEnemy: UnitInstance[];
  validationErrors: string[];      // empty on success
}

export const resolveBattle: (input: ResolverInput) => ResolverOutput;
```

## Steps

1. Build a "battle field": each side's units expanded by `count` into
   individual stat rows with current HP and ability tags from
   `abilities.ts`.
2. Compute initiative order (highest first; ties ‚Üí ordering inside the
   input). Per-card "‚àíN initiative on turn 1" handled.
3. Each tick:
   - The active unit picks a target per its ability (`focus`,
     `splashTargetsMostHealth`, `medicHealsAlly`, ‚Ä¶) and per the enemy's
     `enemyDamageRule`.
   - Apply damage. Splash hits secondary targets at full damage unless
     the ability says otherwise.
   - Special abilities resolve (medic heal, trapper initiative bump, ‚Ä¶).
4. After each "incoming damage" event from enemy ‚Üí player, consume the next
   `DamageAllocation`. Validate:
   - Sum equals incoming damage.
   - Per-unit absorption is lethal OR leftover damage < unit HP.
5. If player runs out of HP ‚Üí `lose`. If enemy out ‚Üí `win`.
6. If `damageAllocations` runs out before resolution ‚Üí `mid`, with
   `validationErrors` populated.

## Ability list (V1)

`focus`, `splash`, `armor` (-1 incoming), `heal`, `singleUse`, `cover`
(redirected from), `ammo` (counts uses), `reload` (skip turn to refresh
uses), `vsBoss¬±N` (modifies attack vs boss), `revealsScout` (-1 enemy init).

## Tests

- 1 Spearman vs 1 Cutter: deterministic outcome, exact log.
- Splash hits two enemies for full damage.
- Armor reduces incoming by 1.
- Medic heals lowest-HP ally.
- Single-use unit drops out after one attack.
- Bad damage allocation: `validationErrors` includes the allocation index
  and reason.
- Mid resolution: outcome is `'mid'` if allocations missing.
