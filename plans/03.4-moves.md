# 03.4 — Engine moves: pullFromMat & supporting wrappers

**Parent:** [03-resources.md](03-resources.md)

## Goal

Surface the resource primitives as bgio moves with stage-aware permission
checks. `chiefDistribute` itself lives in stage 04 since it's chief-specific
behavior, but the *wrappers* the chief move calls live here.

## Files

- Create `src/game/resources/moves.ts`.
- Modify `src/game/moves.ts` to register `pullFromMat`.
- Create `tests/resources/moves.test.ts`.

## API

```ts
import type { Move } from 'boardgame.io';

// Any non-chief seat can pull from their own circle, any time during
// othersPhase or chiefPhase (per game-design.md "as soon as placed").
export const pullFromMat: Move<SettlementState>;
// args: amounts: Partial<ResourceBag>; pulled into a per-player wallet (TBD)
// or directly spent — for now, we put it into a `G.wallets[playerID]` slice.
```

## Steps

1. Add a `wallets: Record<PlayerID, ResourceBag>` slice to `SettlementState`.
   Chief has none (chief acts on the bank directly).
2. `pullFromMat`:
   - Validate `ctx.playerID` is a non-chief seat.
   - Validate the seat has a circle on the mat.
   - Call `pullFromCircle(G.centerMat, playerID, amounts)`.
   - Add the pulled amounts to `G.wallets[playerID]` via `add` from 03.1.
   - Underflow / missing circle → `INVALID_MOVE`.
3. `payFromWallet(G, playerID, cost)` helper used by Science / Domestic /
   Foreign purchase moves — wraps `transfer` from wallet to bank.

## Tests

- Pulling from your own circle deducts from circle and credits wallet.
- Pulling from another seat's circle returns `INVALID_MOVE`.
- Chief seat calling `pullFromMat` returns `INVALID_MOVE` (chief has no
  circle).
- Pulling an amount larger than available returns `INVALID_MOVE` and leaves
  state unchanged.

## Dependencies

01.1, 01.3, 02.1 (so we can gate by phase), 03.1, 03.2, 03.3.

## Out of scope

- `chiefDistribute` (lives in 04.1 — uses the same primitives).
- Per-role purchases (each role's stub wires `payFromWallet`).
