# 12.7 — One-command full-stack dev

**Parent:** [12-testing-debug.md](12-testing-debug.md)

## Goal

Boot the entire local experience — client + bgio server + SQLite +
server-side bots — with a single `npm run dev:full`. Same architecture as
production; just one command instead of two terminals.

We deliberately do **not** relax the "bots run server-side, always" rule
from [11-ai.md](11-ai.md). The dev experience improves through tooling,
not through a parallel client-bot codepath that could drift.

## Files

- Modify `package.json`:
  - Add `concurrently` as devDependency.
  - Add scripts:
    ```
    "dev:full":   "concurrently -n server,client -c blue,green \"npm run server:dev\" \"npm run dev\"",
    "dev:server": "tsx watch server/index.ts",   // alias for server:dev
    ```
- Create `scripts/dev-full.md` (or extend [../README.md](../README.md))
  documenting how to use it and what each color-tagged stream means.
- Modify `.env.example`:
  ```
  VITE_CLIENT_MODE=networked
  VITE_SERVER_URL=http://localhost:8000
  PORT=8000
  STORAGE_KIND=sqlite
  SQLITE_PATH=./.dev-data/settlement.sqlite
  ```
- Add `.dev-data/` to [../.gitignore](../.gitignore).
- Modify [../CLAUDE.md](../CLAUDE.md) "Common commands" to surface
  `npm run dev:full`.

## Steps

1. `concurrently` runs both processes with prefixed output so logs are
   easy to read in one terminal.
2. The dev SQLite file lives at `.dev-data/settlement.sqlite`; nothing in
   git, nothing shared between machines.
3. Hot reload:
   - Client: Vite HMR as today.
   - Server: `tsx watch` restarts on `server/**` or `src/game/**` change
     (same files the production server bundles).
4. Auto-create dev account: a tiny `scripts/dev-seed.ts` (run by
   `dev:full` on first boot when DB is empty) seeds two accounts
   (`alice` / `bob`, password `password`) so login is instant during
   dev. Skipped when DB already has users.
5. Single `Ctrl+C` kills both — `concurrently --kill-others` is the
   default behavior with our flags.

## Tests

- `npm run dev:full` brings up both processes; the client at `:5173`
  successfully reaches the server at `:8000` (smoke test in 12.5
  Playwright already validates the boot path; just point its
  `webServer` at `dev:full` instead of `dev`).
- `dev-seed.ts` is idempotent: second boot does not duplicate accounts.

## Dependencies

10.1 (server package), 10.2 (client modes), 10.4 (storage),
10.7 (accounts — for the dev-seed accounts), 11.7 (server-side solo
bots).

## Out of scope

- Hosting the full stack in a single Node process. Two processes mirrors
  production; no value in collapsing for dev.
- Docker compose. `concurrently` is enough; bringing in Docker adds
  setup cost without payoff at this scale.
- Test-mode auth bypass. Dev-seed accounts log in normally — no
  back-door auth path.
