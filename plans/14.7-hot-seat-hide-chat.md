# 14.7 — Hide chat in hot-seat

**Parent:** [14-playtest-followups.md](14-playtest-followups.md)

## Goal

Chat needs a network transport (10.5 reads `client.chatMessages` and
calls `client.sendChatMessage` — both are bgio multiplayer
features). Hot-seat has no transport, so the composer is dead UI:
"No messages yet." renders forever, sent text goes nowhere — see
playtest issue V8.

## Files

- Modify `src/Board.tsx` — gate `<ChatPane>` and `<ChatComposer>`
  on `clientMode === 'networked'`.

## Steps

1. Read the resolved client mode (from 14.3's `getClientMode()` or
   inline via `detectMode()`).
2. Render the chat pane + composer only when mode is 'networked'.
3. Keep the components themselves untouched — they still work in
   networked mode.

## Tests

- Hot-seat: chat not in DOM (smoke test on a hot-seat client).
- Networked: chat in DOM (already exists from 10.5 testing).

## Dependencies

10.2 (clientMode), 10.5 (chat panes), 14.3 (ideally — but hard
gate is the build-mode flag).

## Out of scope

- Hot-seat "notes to self" feature (a local-only chat-shaped pad).
  Possible follow-up if a player wants to track decisions across
  multi-seat play, but YAGNI for V1.
