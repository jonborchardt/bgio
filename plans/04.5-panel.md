# 04.5 — Chief UI panel

**Parent:** [04-chief.md](04-chief.md)

## Goal

A minimal Chief panel that lets the chief player distribute resources to each
non-chief seat and end their phase. MUI primitives only, theme tokens via
[../src/theme.ts](../src/theme.ts).

## Files

- Create `src/ui/chief/ChiefPanel.tsx`.
- Create `src/ui/chief/CircleEditor.tsx` — small per-target widget.
- Modify `src/Board.tsx` to render `<ChiefPanel>` when local seat holds chief.
- Create `tests/ui/chief/ChiefPanel.test.tsx`.

## API

```tsx
import type { BoardProps } from 'boardgame.io/react';
import type { SettlementState } from '../../game/types';

export const ChiefPanel: React.FC<BoardProps<SettlementState>>;
// renders bank, one CircleEditor per non-chief seat, "End my turn" button
```

## Steps

1. Read state from `props.G.bank`, `props.G.centerMat.circles`,
   `props.G.roleAssignments`.
2. For each non-chief seat:
   - Render a `<CircleEditor seat={...}>` with +/- buttons per resource.
   - On +N or –N, call `props.moves.chiefDistribute(seat, { gold: N })` or
     reverse (if reverse not supported, the UI only allows pushing, never
     pulling back from the circle).
3. "End my turn" button calls `props.moves.chiefEndPhase()`.
4. Disabled states:
   - Buttons grey when `props.ctx.phase !== 'chiefPhase'` or local seat
     isn't chief.
   - +N greyed when bank can't afford.
5. All visual choices route through theme tokens
   (`palette.role.chief`, `palette.resource.gold`, …) — those are added in
   09.4.

## Tests

- Renders without crashing for each `numPlayers ∈ {1,2,3,4}`.
- Clicking "+1 gold" on seat 1 fires `chiefDistribute('1', { gold: 1 })`.
- "End my turn" button is disabled outside `chiefPhase`.

## Dependencies

hardcoded MUI palette colors.
