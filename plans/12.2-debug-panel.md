# 12.2 — bgio Debug panel

**Parent:** [12-testing-debug.md](12-testing-debug.md)

## Goal

Toggle the bgio Debug panel on in dev builds for inspecting `G`, `ctx`,
move history, and triggering arbitrary moves. Off in production.

## Files

- Modify `src/App.tsx` to gate `debug` on `import.meta.env.DEV`.
- Create `tests/debug.test.ts`.

## API

```ts
// In App.tsx
<Client debug={import.meta.env.DEV} ...>
```

## Steps

1. bgio's `Client` accepts a `debug` boolean (or config) — flipping it on
   shows the panel.
2. Suppress in production: `import.meta.env.DEV` is `false` in `vite build`.
3. Add a feature-flagged "expand all" mode so the panel auto-opens to the
   `G` tab in dev (small DX win).

## Tests

- A jsdom mount with `DEV=true` renders the panel root element.
- A jsdom mount with `DEV=false` does not.

## Dependencies

01.3, 09.1.

## Out of scope

- Custom debug widgets beyond the bgio default. Add only if a real need
  appears.
