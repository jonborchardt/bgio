# 08.3 — Promote the four event-move stubs to real moves

**Parent:** [08-cross-cutting.md](08-cross-cutting.md)

## Goal

Replace the stub bodies of `chiefPlayGoldEvent` (04.4),
`sciencePlayBlueEvent` (05.4), `domesticPlayGreenEvent` (06.6), and
`foreignPlayRedEvent` (07.6) with implementations that consult 08.1 / 08.2.
Also adds the `eventResolve` follow-up move for awaiting-input effects.

## Files

- Modify `src/game/roles/{chief,science,domestic,foreign}/play*Event.ts`.
- Create `src/game/events/resolveMove.ts` — the `eventResolve` follow-up.
- Register `eventResolve` in `src/game/moves.ts`.
- Create `tests/events/moves.test.ts`.

## API

```ts
export const eventResolve: Move<SettlementState>;
// args: payload: unknown — typed by the awaiting effect's payloadKind
```

## Steps

1. Each `play*Event` move now:
   - Validates caller seat owns the matching color (chief/gold,
     science/blue, domestic/green, foreign/red).
   - Validates the card is in the seat's hand and not in `used` for the
     current cycle.
   - Validates `playedThisRound[seat]` does not already include the color.
   - Removes the card from hand into `used`.
   - Calls `dispatch(G, ctx, random, card)`.
   - Appends color to `playedThisRound[seat]`.
2. `eventResolve`:
   - Validates seat is in `playingEvent` stage.
   - Looks up the awaited effect from `G._awaitingInput[seat]`.
   - Calls `dispatch` again with the supplied payload to apply.
   - Calls `exitEventStage(events)` to return to the prior stage.

## Tests

- All four move shapes execute end-to-end with a `gainResource` event.
- A `swapTwoScienceCards` event enters `playingEvent`; calling
  `eventResolve({a: 'card-1', b: 'card-2'})` swaps and exits the stage.
- Calling `eventResolve` outside `playingEvent` is `INVALID_MOVE`.
- Wrong-color event in a seat's hand is rejected.

## Dependencies

08.1, 08.2 plus the four play* stubs.

## Out of scope

- Designing the starter event content.
