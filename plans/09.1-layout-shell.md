# 09.1 — Layout shell

**Parent:** [09-ui.md](09-ui.md)

## Goal

The board's outer skeleton: chief controls along the top, three role panels
across the middle, the center mat in the center, and a status / log strip at
the bottom. Visibility per `playerView` already enforced by the engine; the
shell just accommodates "other" panels being read-only when collapsed.

## Files

- Modify `src/Board.tsx` to render the shell.
- Create `src/ui/layout/BoardShell.tsx`.
- Create `src/ui/layout/RoleSlot.tsx` — one slot per role; chooses
  expanded / collapsed mode.
- Create `src/ui/layout/StatusBar.tsx` — current phase, current player,
  round counter.
- Create `tests/ui/layout/BoardShell.test.tsx`.

## API

```tsx
export interface BoardShellProps {
  chief: React.ReactNode;
  science: React.ReactNode;
  domestic: React.ReactNode;
  foreign: React.ReactNode;
  centerMat: React.ReactNode;
  status: React.ReactNode;
}
export const BoardShell: React.FC<BoardShellProps>;
```

## Steps

1. Use MUI `Grid` (or `Box` with CSS Grid) for the layout:
   ```
   ┌──────────────── chief ────────────────┐
   │ science │   centerMat   │   domestic  │
   │         │               │             │
   │         │   foreign     │             │
   └─────────────── status ────────────────┘
   ```
2. `RoleSlot` props: `expanded?: boolean`. Collapsed mode renders a tiny
   summary; expanded renders the role's panel.
3. `Board.tsx` decides expansion: a role is expanded if the local seat
   holds it, or `numPlayers === 1` (hot-seat solo: everything expanded).
4. `StatusBar` reads `props.ctx.phase`, `props.ctx.currentPlayer`,
   `props.G.round`.

## Tests

- 4-player game, local seat is Foreign: only the Foreign slot is expanded;
  others render their summary view.
- 1-player game: every slot is expanded.
- Status bar shows the right phase / player / round on a fresh game.

## Dependencies

01, 02. Independent of role panels (they're injected via props).

## Out of scope

- Mobile layout. V1 is desktop-only.
