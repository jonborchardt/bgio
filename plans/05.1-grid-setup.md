# 05.1 — Science grid setup (Option 1, 3×3)

**Parent:** [05-science.md](05-science.md)

## Goal

Build the 3×3 grid of science cards at game start, with three tiers of
9 cards each (beginner / intermediate / advanced) and 4 random tech cards
slotted under each science card per game-design.md §Science Option 1.

## Files

- Create `src/data/scienceCards.json` — content file (initially seeded with
  cost templates; real content lives in JSON).
- Create `src/data/scienceCards.ts` — typed loader (mirrors 01.2 pattern).
- Create `src/game/roles/science/setup.ts`.
- Modify `src/game/setup.ts` to call science setup.
- Create `tests/roles/science/setup.test.ts`.

## API

```ts
export interface ScienceCardDef {
  id: string;
  tier: 'beginner' | 'intermediate' | 'advanced';
  color: 'red' | 'gold' | 'green' | 'blue';
  level: 0 | 1 | 2 | 3;            // for "lowest first" rule
  cost: ResourceBag;
  prereqIDs?: string[];             // optional cross-references
}

export interface ScienceState {
  grid: ScienceCardDef[][];                 // 3 columns × 3 rows
  underCards: Record<string, TechnologyDef[]>; // by science card id
  paid: Record<string, ResourceBag>;        // contributions per card
  completed: string[];                       // ids
  perRoundCompletions: number;               // resets at endOfRound
}

export const setupScience: (random: RandomAPI) => ScienceState;
```

## Steps

1. `scienceCards.json` defines 27 beginner + 36 intermediate + 27 advanced
   = 90 cards total per game-design.md §Card Counts.
2. Each card has a `tier`, `color`, `level`, and `cost`. Initially `cost`
   is a flat resource sum (Option 1).
3. `setupScience`:
   - shuffle each tier separately,
   - take 9 from each tier,
   - lay out 3×3 grouping by color in columns, lowest level in row 0
     (closest to player), per game-design.md.
   - For each cell, draw 4 tech cards from `TECHNOLOGIES` whose color and
     level match; rest go to a "discard"/box.
4. Register a round-end hook `science:reset-completions` (via 02.5) that
   clears `perRoundCompletions = 0`.

## Tests

- Setup is deterministic given a seed (same grid IDs).
- Each column shares a single color; row 0 has the lowest level in that
  color.
- Each cell has exactly 4 cards under it whose color matches the science
  card's color.
- Tier counts: 3 cards from each tier appear in the grid (one per column,
  one per row layer).
- Round-end hook resets `perRoundCompletions` after the first round.

## Dependencies

01.2 (data loaders), 02.3 (random), 02.5 (hooks), 03 (resources for cost
typing).

## Out of scope

- Option 1b / 2 / 3 alternatives.
- Tech card *effects* (consumed by other roles when distributed).
