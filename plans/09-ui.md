# 09 — UI

## Goal

Per-role panels, a center-mat view, reusable card components, and a coherent
MUI theme — so the gameplay model from 02–08 actually plays.

## Scope

**In:**
- Layout: a board with a slot per non-chief role panel + the center mat in
  the middle + the chief's controls along the top. Hot-seat: every panel
  visible (or collapsible per `playerView`). Networked: only your panel is
  expanded; others are read-only summaries.
- Reusable components:
  - `<ResourceBag>` — typed resource list with theme tokens per resource.
  - `<CenterMat>` — circles + trade slot.
  - `<BuildingCard>`, `<UnitCard>`, `<ScienceCard>`, `<EventCard>`,
    `<TechCard>` — driven from JSON data, no hardcoded strings.
  - `<DeckStack>` — face-down with count + face-up top.
- Theme work in [src/theme.ts](../src/theme.ts):
  - Per-resource ramp slots + semantic tokens (`palette.resource.gold`, …).
  - Per-role accent ramp + semantic token (`palette.role.chief`, etc.).
  - Card surface tokens for tier (beginner / intermediate / advanced).

**Out:**
- A "polished" / playtest-ready visual pass — that is its own sub-plan after
  gameplay is stable.
- Animations.
- Mobile-specific layouts.

## Depends on

01, 02, 03 — and ideally per-role state from 04–07. Each role stub already
includes an `NN-role/*-panel.md` sub-plan that depends on this stage's
shared components and theme tokens.

## Test surface

- Pure component tests with React Testing Library: cards render every
  attribute pulled from JSON without any hardcoded labels.
- Theme contract test: every documented palette key resolves to a non-empty
  hex.
- Visual sanity: a stage-12 sub-plan adds Playwright snapshot of the empty
  board so style regressions are caught.
