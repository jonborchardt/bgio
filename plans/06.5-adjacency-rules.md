# 06.5 — Adjacency-effect registry

**Parent:** [06-domestic.md](06-domestic.md)

## Goal

A pluggable registry that lets us say "if Building X is adjacent to
Building Y, modify produce yield by Z" without hardcoding pairs into
domestic/produce.ts. Adjacency is gameplay-relevant in V1, not cosmetic —
this sub-plan ships the registry; [06.8](06.8-adjacency-content.md) ships
the initial content.

## Files

- Create `src/game/roles/domestic/adjacency.ts`.
- Modify `src/game/roles/domestic/produce.ts` to consult the registry.
- Create `tests/roles/domestic/adjacency.test.ts`.

## API

```ts
export interface AdjacencyRule {
  /** When this `defID` has a neighbor whose defID matches `whenAdjacentTo` */
  defID: string;
  whenAdjacentTo: string | '*';   // '*' = any building
  bonus: Partial<ResourceBag>;     // added to produce yield, once per match
}

export const adjacencyRules: AdjacencyRule[]; // initially empty

export const yieldAdjacencyBonus = (
  grid: Map<string, DomesticBuilding>,
  rules: ReadonlyArray<AdjacencyRule>,
): ResourceBag;
```

## Steps

1. `yieldAdjacencyBonus` walks each cell, finds neighbors via cellKey
   adjacency, applies any matching rule, sums bonuses.
2. `produce` (06.4) calls `yieldAdjacencyBonus(grid, adjacencyRules)` and
   adds the result to the total yield.
3. Rules can be added at content time — JSON file or tests can extend
   `adjacencyRules` via a registry function (avoid mutating module-level
   array directly outside tests; provide
   `__setAdjacencyRulesForTest(rules)`).

## Tests

- Empty registry: `yieldAdjacencyBonus` returns `EMPTY_BAG`.
- Two adjacent matching buildings yield the bonus once per neighbor pair
  per rule.
- Non-adjacent buildings: no bonus.
- `'*'` rule: any building neighbor triggers.
