Tier 4: server / infra hardening sweep

Closes orchestrator Tier 4 — twelve medium/low items. Issue 054 (log
table compaction) deferred — needs a retention-policy decision.

- 022: shared `server/storage/migrate.ts` adds a `_migrations` ledger
  + EXCLUSIVE-locked apply loop. Both bgio match adapter and the
  auth/runs SQLite stores route migrations through it, so a second
  boot skips applied files and concurrent boots can't double-apply.
- 023: rate-limit `buckets` Map capped at 10k entries with FIFO
  eviction so a flood of unique attacker IPs can't grow the table
  unbounded. New `TRUST_PROXY=true` env opt-in flips the Koa app's
  proxy flag so `ctx.ip` reads X-Forwarded-For (without it, every
  attacker behind a proxy collapses into one bucket).
- 024: deploy-server.yml now runs `vitest tests/server` and
  `npm run lint` after typecheck — Render no longer auto-deploys
  when server tests or lint are red.
- 044: `start.ts` (and the new `bots/`) added to server/tsconfig.json
  include, so server:build typechecks the entrypoints.
- 045: Dockerfile gains HEALTHCHECK + USER node + a node-direct CMD
  (skips the per-restart `npx` lookup tax).
- 050: .env.example expanded; SQLITE_PATH default unified on
  `./bgio-data/settlement.sqlite` across server/README.md,
